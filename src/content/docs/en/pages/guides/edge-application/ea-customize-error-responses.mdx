Configure custom error pages that are delivered when requests return 4xx or 5xx HTTP status codes. There are separate instructions for the new [Custom Pages](/en/documentation/products/secure/custom-pages/) and the legacy [Error Responses](/en/documentation/products/build/edge-application/error-responses/).

:::tip
If you're not sure which steps apply to your account, see [this guide](/en/documentation/products/guides/verify-account-migration/) to determine if your account has already been migrated.
:::

## Customizing error pages

<Fragment slot="panel.consoleworkloads">

To configure Custom Pages:

1. Access [Azion Console](/en/documentation/products/guides/how-to-access-azion-console/) > **Workloads**.
2. Select the workload for your error pages.
3. Navigate to **Deployment Settings**.
4. In **Custom Pages**, click **+ Custom Page** or select an existing one.
5. Give your Custom Page a name.
6. Configure the **Status Code** you want to customize.
7. Set the **Response TTL** in seconds for cache duration.
8. In **Page Path (URI)**, add the path to your custom error page.
9. Select the appropriate **Edge Connector** to serve the error page.
10. Optionally, customize the **Response Status Code** that users will receive.
11. Click **Save**.

</Fragment>

<Fragment slot="panel.consoledomain">

To configure Error Responses:

1. Access [Azion Console](/en/documentation/products/guides/how-to-access-azion-console/) > **Edge Application**.
2. Select the application linked to the domain that needs the new error page.
3. Select the **Error Responses** tab.
4. The first **Status Code** field is disabled, but in **Default Response TTL** you can set the cache *time-to-live* (TTL), in seconds, of all error pages.
    - This first status code serves as a placeholder to cache the page in case an unexpected status is received.
5. To customize specific error codes, click the **+ Add Error Response** button.
6. In the new **Status Code** field, select the status code you want to customize.
7. Change the cache TTL in the **Custom Response TTL** field.
   - You can keep the same cache TTL set in the previous field or add a different TTL.
8. In the **Page Path** field, you can add a *path* to an error page created in the source.
9. In the **Response Status Code** field, you can customize the HTTP status that will be received by the user.
10. In the **Set Origin** section, you can define the origin of error page URIs.
    - You can configure their origins in the application's **Origins** tab.
11. Click the **Save** button.

<Fragment slot="panel.apiv4">

1. Run the following `POST` request to create a Custom Page, replacing `[TOKEN VALUE]` with your [personal token](/en/documentation/products/guides/personal-tokens/):

<Code lang="bash" code={`curl --request POST \\
  --url https://api.azion.com/v4/workspace/custom_pages \\
  --header 'Accept: application/json' \\
  --header 'Authorization: Token [TOKEN VALUE]' \\
  --header 'Content-Type: application/json' \\
  --data '{
    "name": "404 error page",
    "active": true,
    "pages": [{
      "code": "404",
      "page": {
        "type": "page_connector",
        "attributes": {
          "connector": <edge_connector_id>,
          "ttl": 300,
          "uri": "/error/404.html",
          "custom_status_code": 404
        }
      }
    }]
  }'`} />

2. You'll receive a response with the Custom Page details.

3. Bind the Custom Page to a Workload Deployment:

<Code lang="bash" code={`curl --request POST \\
  --url https://api.azion.com/v4/workspace/workloads/<workload_id>/deployments \\
  --header 'Accept: application/json' \\
  --header 'Authorization: Token [TOKEN VALUE]' \\
  --header 'Content-Type: application/json' \\
  --data '{
    "name": "Workload Deployment with Custom Pages",
    "current": true,
    "active": true,
    "strategy": {
      "type": "default",
      "attributes": {
        "edge_application": <edge_application_id>,
        "custom_page": <custom_page_id>
      }
    }
  }'`} />

:::tip
Check the [Custom Pages API documentation](https://api.azion.com/v4#/operations/GetWorkspaceCustomPages) for more details.
:::

</Fragment>


</Tabs>

Done. Your application will now display custom error pages with the new configuration.
