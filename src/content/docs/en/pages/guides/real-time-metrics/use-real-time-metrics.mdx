**Real-Time Metrics** provides real-time access to metrics through charts. Charts exhibit data as soon as your applications and other products begin to have incoming accesses and traffic.

:::tip
If you're not sure which steps apply to your account, see [this guide](/en/documentation/products/guides/verify-account-migration/) to determine if your account has already been migrated.
:::

<LinkButton link="/en/documentation/products/observe/real-time-metrics/" label="go to Real-Time Metrics reference" severity="secondary" />

:::note
The new Real-Time Metrics provides data and metrics starting from **October 15th, 2022**. If you want to view metrics for up to 2 years and before October 15th, 2022, see [Historical Real-Time Metrics](/en/documentation/products/observe/historical-real-time-metrics/).
:::

<Fragment slot="panel.consoleworkloads">

To access Real-Time Metrics via Workloads:

1. Access [Azion Console](https://console.azion.com) > **Workloads**.
2. Select your workload.
3. Navigate to **Monitoring** tab.
4. View real-time metrics and charts for your workload:
   - **Requests**: Request count and response times
   - **Data Transfer**: Bandwidth usage
   - **Status Codes**: HTTP response codes distribution
   - **Security**: WAF events and firewall actions

:::note
With Workloads, metrics are automatically filtered for your specific workload configuration.
:::

</Fragment>

Access [Azion Console](https://console.azion.com) and select **Products menu** > **Real-Time Metrics** on the **Observe** section.

<Fragment slot="panel.apiv4">

To access metrics via API v4:

<Code lang="bash" code={`curl --request GET \\\\
  --url https://api.azion.com/v4/metrics/edge_applications \\\\
  --header 'Accept: application/json' \\\\
  --header 'Authorization: Token [TOKEN VALUE]' \\\\
  --data '{
    "start_time": "2024-01-01T00:00:00Z",
    "end_time": "2024-01-01T23:59:59Z",
    "granularity": "hour",
    "fields": ["requests", "data_transferred", "status_codes"]
  }'`} />

Available metrics endpoints:
- `/v4/metrics/edge_applications`
- `/v4/metrics/waf`
- `/v4/metrics/edge_functions`
- `/v4/metrics/data_stream`

:::tip
Check the [Metrics API v4 documentation](https://api.azion.com/v4#/operations/GetMetricsEdgeApplications) for more details.
:::

</Fragment>

1. Select one of the three available categories on the dropdown menu:

    <Tag severity="info" client:only="vue">Build</Tag> <Tag severity="info" client:only="vue">Secure</Tag> <Tag severity="info" client:only="vue">Observe</Tag>

2. Select a tab according to the product you want to view:

-  **Build**

    <Tag severity="info" client:only="vue">Edge Application</Tag> <Tag severity="info" client:only="vue">Tiered Cache</Tag> <Tag severity="info" client:only="vue">Edge Functions</Tag> <Tag severity="info" client:only="vue">Image Processor</Tag>

- **Secure**

    <Tag severity="info" client:only="vue">WAF</Tag> <Tag severity="info" client:only="vue">Edge DNS</Tag>

- **Observe**

    <Tag severity="info" client:only="vue">Data Stream</Tag>

3. On **Time range**, select a time period from the options to fetch the data that'll be exhibited on the charts:

    <Tag severity="info" client:only="vue">Last Hour</Tag> <Tag severity="info" client:only="vue">Last 24 Hours</Tag> <Tag severity="info" client:only="vue">Last 7 Days</Tag> <Tag severity="info" client:only="vue">Last 30 Days</Tag> <Tag severity="info" client:only="vue">Last 6 Months</Tag>

4. If you want to use a different date from the options, click the calendar fields and select a beginning and ending date and time.
5. If you've selected the **Edge Applications** product tab, select between one of the four subtabs:

    <Tag severity="info" client:only="vue">Data Transferred</Tag> <Tag severity="info" client:only="vue">Requests</Tag> <Tag severity="info" client:only="vue">Status Codes</Tag> <Tag severity="info" client:only="vue">Bandwidth Saving</Tag>

All charts from all tabs are automatically refreshed after applying a time range.
