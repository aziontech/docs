Este glossário apresenta os principais conceitos e terminologia relacionados à plataforma e aos produtos Azion.

Para obter mais informações sobre terminologia e conceitos de edge, consulte [State of the Edge](https://stateoftheedge.com/). Para a terminologia de tecnologia em geral, consulte [NIST](https://www.nist.gov/).

### ANAME

O registro de tipo de nome de *domínio raiz*, também conhecido como *naked domain* registrado no [DNS](#dns). Este é o domínio que você adquire e gerencia de um provedor de domínio externo. Por exemplo: `yourdomain.org`.

---

### API

Application Programming Interface (API). Um conjunto de especificações que fornece maneiras para diferentes programas se comunicarem uns com os outros. Consulte a [Azion API](https://api.azion.com/).

---

### Arquitetura cliente servidor

Modelo em que a carga de trabalho e as responsabilidades de um sistema são divididas entre dois tipos de entidades: clientes e servidores. Os clientes são os dispositivos ou aplicações do usuário final que solicitam serviços ou recursos dos servidores. Os servidores são os sistemas centralizados que fornecem esses serviços ou recursos aos clientes.

---

### Azion SDK

Um kit de ferramentas para desenvolvedores que simplifica o processo de interação com os serviços de API da Azion ao desenvolver aplicativos na linguagem de programação Go. Torna mais fácil para os desenvolvedores do Go integrar os serviços da Azion em seus projetos. [Veja Azion SDK](/pt-br/documentacao/devtools/sdk/go/).

---

### Bandwidth

Quantidade de informações recebidas a cada segundo. Na Azion, isso representa o conteúdo que foi entregue pelas suas aplicações a cada segundo.

---

### Cache key

Uma entrada de índice para um objeto no cache da Azion.

---

### Certificado

Documento digital que verifica a identidade de uma entidade, como um site ou organização, e estabelece uma conexão segura entre a entidade e o dispositivo do usuário. É usado no contexto de comunicação segura pela internet, especificamente no caso de conexões HTTPS (Hypertext Transfer Protocol Secure). Os certificados garantem que os dados transmitidos entre o dispositivo do usuário e o servidor da entidade sejam criptografados e não possam ser interceptados ou adulterados por agentes mal-intencionados.

---

### Cipher

Método ou algoritmo usado para criptografar ou codificar informações para mantê-las seguras e confidenciais. Envolve o processo de conversão de texto simples (dados não criptografados) em texto cifrado (dados criptografados) usando um conjunto específico de regras ou operações matemáticas. As cifras podem ser classificadas em vários tipos, como cifras de substituição, cifras de transposição, cifras de chave simétrica e cifras de chave assimétrica.

---

### CLI

Interface que permite aos usuários interagir com aplicativos e sistemas via linha de comando. [Veja Azion CLI](/pt-br/documentacao/produtos/azion-cli/visao-geral/).

---

### CNAME

O registro de tipo de *nome comum* de um domínio registrado no DNS. Ao usar o [Intelligent DNS](/pt-br/documentacao/produtos/intelligent-dns/), você pode criar e gerenciar registros CNAME sem usar um provedor de DNS.

---

### Composable apps

As [composable applications](https://www.azion.com/pt-br/blog/composable-applications-o-que-sao-e-por-que-voce-precisa-delas/) são construídas reutilizando blocos, pedaços de código ou soluções existentes, como se fossem módulos. Com esse paradigma, é possível reutilizar componentes, sem a necessidade de desenvolver a partir do zero e dando mais agilidade, inovação e adaptabilidade.

---

### Conformidade

Conformidade (compliance) refere-se ao cumprimento de todas as normas e especificações legais do setor, incluindo requisitos regulamentares, políticas e leis.

---

### Continuous deployment

Prática de desenvolvimento de software que se refere à liberação automática de novas alterações e recursos em produção, após as compilações terem concluído com sucesso todo o ciclo de testes definido. Agiliza a entrega de funcionalidades aos clientes e também reduz o risco de introdução de bugs ou problemas por causa dos testes regulares no código.

---

### Cookie

Pequeno pedaço de dados que é armazenado no computador de um usuário por um site. Ele é enviado pelo site para o navegador do usuário e, em seguida, é armazenado no computador do usuário. Os cookies são comumente usados para armazenar informações sobre as preferências do usuário, status de login e atividade de navegação em um site. Eles são frequentemente usados para fins de [autenticação](#autenticação) e personalização. Os cookies podem ser cookies de sessão, que são temporários e são excluídos quando o usuário fecha o navegador, ou cookies persistentes, que são armazenados no computador do usuário por um longo período de tempo.

---

### CRUD

Criar, ler, atualizar e excluir (CRUD). As quatro funções básicas e necessárias para os modelos de computação.

---

### Dispositivo

Equipamentos eletrônicos, como um celular ou um computador.

---

### Distributed Denial of Service (DDoS)

Este ataque é um tipo de crime cibernético que visa interromper a disponibilidade de um site, rede ou aplicativo, sobrecarregando-o com uma enorme quantidade de tráfego falso. O objetivo do ataque é fazer com que os sistemas-alvo fiquem inacessíveis para usuários legítimos. Para combater esse tipo de ataque, a Azion oferece [proteção DDoS](https://www.azion.com/pt-br/solucoes/ddos-protection/).

---

### DNS

Domain Name System (DNS). Um sistema hierárquico de nomes que traduz nomes de domínio legíveis por humanos em endereços IP (Internet Protocol) que os computadores usam para identificar uns aos outros na rede. O DNS permite que os usuários acessem sites e serviços da web usando nomes de domínio, em vez de ter que memorizar endereços IP numéricos complexos. Por exemplo, o DNS traduz o nome de domínio "www.example.com" no endereço IP correspondente, permitindo que um navegador da web alcance o servidor da web do site example.com.

---

### Edição de código em tempo real

Possibilidade de editar o código do seu site ou aplicativo em tempo real, sem a necessidade de parar ou interromper a aplicação.

---

### Edge

Os servidores ou pontos de presença de rede localizados na borda da rede, mais próximos dos usuários finais, geralmente nas localizações geográficas onde a entrega de conteúdo é necessária. A computação na borda (edge computing) envolve processar dados e executar aplicativos mais perto dos usuários finais, em vez de depender de servidores centralizados em data centers distantes.

---

### Edge Application

Aplicativo executado na borda da rede, em servidores ou pontos de presença próximos aos usuários finais. Isso ajuda a melhorar o desempenho e a latência de aplicativos, pois reduz a distância física entre os servidores e os dispositivos dos usuários.

---

### Edge Computing

Uma abordagem de computação que envolve processar dados e executar aplicativos mais perto dos usuários finais, em vez de depender de servidores centralizados em data centers distantes. Isso ajuda a reduzir a latência e melhorar o desempenho de aplicativos, especialmente para aqueles que requerem respostas rápidas em tempo real.

---

### Encryption

O processo de conversão de informações em um formato ilegível, chamado texto cifrado, para proteger a confidencialidade e a segurança dos dados. Apenas as partes autorizadas com a chave de descriptografia correta podem reverter o texto cifrado para seu formato original e legível.

---

### Enforceability

A capacidade de fazer cumprir ou executar uma política ou regulamento. Em um contexto de segurança de informações, refere-se à eficácia das medidas e controles implementados para garantir que as políticas de segurança sejam seguidas e que as violações sejam detectadas e tratadas adequadamente.

---

### Event-driven architecture

Uma arquitetura de software em que os componentes do sistema respondem a eventos gerados por outros componentes, em vez de seguir um fluxo de controle de programa rígido. Isso permite que os sistemas sejam mais flexíveis e responsivos às mudanças nas condições e no ambiente.

---

### GCP

Google Cloud Platform (GCP). Uma plataforma de computação em nuvem oferecida pelo Google que fornece uma variedade de serviços de infraestrutura, armazenamento, análise, aprendizado de máquina e desenvolvimento de aplicativos para empresas e desenvolvedores.

---

### Go

Linguagem de programação Go, também conhecida como Golang. Uma linguagem de programação de código aberto criada pelo Google que é conhecida por sua eficiência e desempenho, especialmente em sistemas distribuídos e aplicativos de rede.

---

### GraphQL

Uma linguagem de consulta para suas APIs e um tempo de execução para executar essas consultas, permitindo que os clientes solicitem exatamente os dados de que precisam, sem mais nem menos. Isso ajuda a reduzir o consumo de largura de banda e a melhorar o desempenho das APIs.

---

### HSM

Hardware Security Module (HSM). Um dispositivo de hardware dedicado que fornece segurança adicional para operações criptográficas e proteção de chaves de criptografia. Os HSMs são frequentemente usados em ambientes de segurança de informações para armazenar chaves de criptografia sensíveis e executar operações criptográficas de forma segura.

---

### Hybrid Cloud

Uma combinação de infraestrutura de nuvem pública e privada que permite que as empresas aloquem recursos de acordo com suas necessidades específicas e mantenham o controle sobre dados e aplicativos críticos, ao mesmo tempo em que aproveitam a escalabilidade e a flexibilidade da nuvem pública.

---

### Hypertext Transfer Protocol Secure (HTTPS)

Uma extensão do HTTP (Hypertext Transfer Protocol) que adiciona uma camada de segurança por meio da criptografia. O HTTPS é amplamente usado para proteger a comunicação entre navegadores da web e servidores da web, garantindo que os dados transmitidos sejam confidenciais e não possam ser interceptados ou adulterados por agentes mal-intencionados.

---

### IaaS

Infrastructure as a Service (IaaS). Um modelo de serviço de nuvem que fornece acesso sob demanda a recursos de infraestrutura de TI, como servidores virtuais, armazenamento e rede. Os clientes podem provisionar e gerenciar esses recursos de forma flexível, sem a necessidade de comprar e manter hardware físico.

---

### Intelligent Edge

A capacidade de processar dados e executar aplicativos diretamente na borda da rede, mais próximos dos usuários finais, aproveitando a computação na borda para melhorar o desempenho, a latência e a eficiência.

---

### Intelligent Platform

Uma plataforma de computação na borda (edge computing platform) que permite a execução de aplicativos e serviços diretamente na borda da rede, mais próximos dos usuários finais. Isso ajuda a melhorar o desempenho, a latência e a eficiência das aplicações.

---

### IPv4

Internet Protocol version 4 (IPv4). A quarta versão do protocolo de internet, que usa endereços IP de 32 bits. O IPv4 é amplamente utilizado, mas está se esgotando devido ao rápido crescimento da internet, o que levou à adoção do IPv6.

---

### IPv6

Internet Protocol version 6 (IPv6). A sexta versão do protocolo de internet, que usa endereços IP de 128 bits. O IPv6 foi desenvolvido para substituir o IPv4 e fornecer um número muito maior de endereços IP, a fim de atender às crescentes necessidades da internet.

---

### ISO 27001

ISO/IEC 27001. Uma norma internacional para segurança da informação que estabelece requisitos e práticas recomendadas para o gerenciamento de segurança de informações em organizações. A conformidade com o ISO 27001 demonstra um compromisso com a segurança da informação e a proteção dos ativos da organização.

---

### ITIL

IT Infrastructure Library (ITIL). Um conjunto de práticas recomendadas para o gerenciamento de serviços de TI (TSM) que se concentra na alinhamento de serviços de TI com as necessidades do negócio. O ITIL fornece um conjunto abrangente de processos e procedimentos para melhorar a eficiência e a eficácia do fornecimento de serviços de TI.

---

### Identity and Access Management (IAM)

Gerenciamento de Identidade e Acesso (IAM). Um conjunto de práticas, políticas e tecnologias para gerenciar identidades digitais, controlar o acesso a recursos de TI e garantir a segurança da informação. O IAM envolve autenticação, autorização e gerenciamento de privilégios.

---

### Internet of Things (IoT)

Internet das Coisas (IoT). Uma rede de dispositivos físicos, veículos, eletrodomésticos e outros objetos incorporados com sensores, software, conectividade e eletrônicos que permitem que esses objetos coletem e troquem dados. A IoT permite a automação e a coleta de dados em tempo real em uma variedade de aplicações.

---

### JSON

JavaScript Object Notation (JSON). Um formato leve de troca de dados que é fácil para os humanos lerem e escreverem, e fácil para as máquinas analisarem e gerarem. O JSON é comumente usado para estruturar dados que são transmitidos entre um servidor e um cliente, ou entre diferentes partes de um aplicativo.

---

### Java

Linguagem de programação Java. Uma linguagem de programação de alto nível conhecida por sua portabilidade, o que significa que os programas escritos em Java podem ser executados em diferentes sistemas operacionais sem a necessidade de alterações significativas. A Java é amplamente utilizada no desenvolvimento de aplicativos empresariais, aplicativos móveis e na criação de applets da web.

---

### KPI

Key Performance Indicator (KPI). Um indicador chave de desempenho é uma métrica quantitativa usada para avaliar o desempenho de uma organização, equipe ou processo em relação a metas e objetivos específicos.

---

### Kubernetes

Um sistema de orquestração de contêiner de código aberto amplamente utilizado para automatizar a implantação, o dimensionamento e o gerenciamento de aplicativos em contêiner. O Kubernetes é projetado para simplificar e acelerar o processo de implantação e operação de aplicativos em contêiner em grande escala.

---

### Lambda Architecture

Uma arquitetura de processamento de dados em tempo real que combina processamento em lote e em tempo real para fornecer análises em tempo real de dados em grande escala. A Lambda Architecture é frequentemente usada em sistemas de Big Data para lidar com grandes volumes de dados de maneira eficiente.

---

### Machine Learning

Aprendizado de máquina (Machine Learning, ML). Um campo da inteligência artificial que se concentra no desenvolvimento de algoritmos e modelos que permitem que os sistemas de computador melhorem seu desempenho em tarefas específicas à medida que adquirem experiência, sem serem explicitamente programados. O aprendizado de máquina é amplamente usado em uma variedade de aplicativos, desde reconhecimento de padrões até previsões e recomendações.

---

### Microservices

Microservices, ou arquitetura de microsserviços, é um estilo arquitetural que estrutura uma aplicação como uma coleção de serviços independentes e interconectados. Cada serviço é responsável por executar uma função específica e se comunica com outros serviços por meio de APIs. Essa abordagem permite que aplicativos sejam altamente escaláveis, flexíveis e fáceis de manter.

---

### Mobile App Development

O desenvolvimento de aplicativos móveis refere-se ao processo de criação de software especificamente para dispositivos móveis, como smartphones e tablets. Isso geralmente envolve o desenvolvimento de aplicativos para plataformas como Android e iOS, que têm requisitos técnicos e de design distintos.

---

### Nginx

Um servidor web e proxy reverso de código aberto amplamente utilizado. O Nginx é conhecido por sua alta performance, escalabilidade e recursos avançados de proxy reverso, o que o torna uma escolha popular para servir conteúdo da web e equilibrar carga entre servidores.

---

### NoSQL

NoSQL (not only SQL) refere-se a uma categoria de sistemas de gerenciamento de banco de dados que não são baseados na linguagem SQL tradicional. Os bancos de dados NoSQL são projetados para lidar com volumes de dados muito grandes e variados e são frequentemente usados em aplicativos web e móveis, bem como em sistemas de Big Data.

---

### OAuth

OAuth (Open Authorization). Um protocolo de autorização que permite que aplicativos de terceiros acessem com segurança os recursos de um usuário sem ter que saber suas credenciais. O OAuth é comumente usado para permitir que os usuários façam login em aplicativos ou sites usando suas credenciais de uma conta existente, como sua conta do Google ou do Facebook.

---

### Open Source

Open source (código aberto) refere-se a software cujo código-fonte é disponibilizado ao público para visualização, uso, modificação e distribuição. Isso permite que uma comunidade de desenvolvedores colabore na melhoria e manutenção do software.

---

### PaaS

Platform as a Service (PaaS). Um modelo de serviço de nuvem que fornece uma plataforma completa de desenvolvimento e implantação de aplicativos, incluindo ferramentas, infraestrutura e serviços para ajudar os desenvolvedores a criar, testar e implantar aplicativos de maneira eficiente.

---

### PCI DSS

Payment Card Industry Data Security Standard (PCI DSS). Um conjunto de padrões de segurança de dados desenvolvido para proteger as informações do cartão de pagamento e garantir a segurança das transações com cartão de crédito e débito. O PCI DSS estabelece requisitos para organizações que lidam com informações de pagamento.

---

### REST

Representational State Transfer (REST). Um estilo arquitetural comumente usado na construção de serviços web. Os serviços RESTful são projetados para serem simples, escaláveis e funcionar de forma confiável na web. Eles usam um conjunto de operações HTTP (como GET, POST, PUT e DELETE) para manipular recursos, que são representados por URLs.

---

### Role-based Access Control (RBAC)

Controle de Acesso Baseado em Função (RBAC). Um modelo de controle de acesso que atribui permissões e privilégios aos usuários com base em suas funções e responsabilidades na organização. O RBAC ajuda a garantir que os usuários tenham acesso apenas aos recursos necessários para realizar suas tarefas.

---

### Scalability

Escalabilidade refere-se à capacidade de um sistema, aplicativo ou serviço de se adaptar ao aumento da demanda ou carga sem perder desempenho ou disponibilidade. A escalabilidade é fundamental para atender às necessidades de crescimento de uma organização.

---

### Serverless

Serverless é um modelo de computação em nuvem que permite que os desenvolvedores criem e executem aplicativos sem a necessidade de gerenciar servidores ou infraestrutura. Em vez disso, os provedores de nuvem gerenciam automaticamente a infraestrutura, permitindo que os desenvolvedores se concentrem apenas na lógica do aplicativo.

---

### Single Sign-On (SSO)

Single Sign-On (SSO). Um sistema de autenticação que permite que os usuários façam login em vários aplicativos e serviços com um único conjunto de credenciais. Isso simplifica o processo de login e melhora a experiência do usuário.

---

### SQL

Structured Query Language (SQL). Uma linguagem de programação usada para gerenciar e manipular bancos de dados relacionais. O SQL é amplamente utilizado em sistemas de gerenciamento de banco de dados (DBMS) para realizar operações como consultas, inserções, atualizações e exclusões de dados.

---

### SSL

Secure Sockets Layer (SSL). Um protocolo de segurança que garante a criptografia de dados transmitidos entre um navegador da web e um servidor da web, garantindo que os dados não possam ser interceptados ou adulterados por agentes mal-intencionados. O SSL é comumente usado em conexões HTTPS para garantir a segurança da comunicação na web.

---

### SaaS

Software as a Service (SaaS). Um modelo de serviço de nuvem que fornece acesso a aplicativos de software hospedados na nuvem, em vez de exigir que os usuários instalem e gerenciem o software em seus próprios dispositivos. Os aplicativos SaaS são acessados por meio de um navegador da web e geralmente são oferecidos sob um modelo de assinatura.

---

### Threat Intelligence

Inteligência de Ameaças. Informações e análises sobre ameaças cibernéticas, incluindo ameaças emergentes, vulnerabilidades de segurança e ataques recentes. A inteligência de ameaças ajuda as organizações a entender e mitigar riscos de segurança.

---

### User Experience (UX)

Experiência do Usuário (UX). A experiência geral de um usuário ao interagir com um produto ou serviço, incluindo a facilidade de uso, a eficiência, a satisfação e a qualidade geral da interação. O design de UX visa melhorar a experiência do usuário por meio de design centrado no usuário e pesquisa.

---

### Virtual Private Cloud (VPC)

Virtual Private Cloud (VPC). Uma rede virtual isolada e privada que permite que as organizações criem ambientes de computação em nuvem seguros e personalizados. As VPCs são frequentemente usadas para isolar recursos e aplicativos em nuvem e fornecer uma camada adicional de segurança.

---

### VPN

Virtual Private Network (VPN). Uma rede privada virtual que permite que os usuários se conectem a uma rede segura através de uma conexão criptografada, mesmo quando estão fora da rede física. As VPNs são comumente usadas para acesso remoto seguro a recursos de rede.

---

### Web Application Firewall (WAF)

Web Application Firewall (WAF). Um firewall projetado especificamente para proteger aplicativos web contra ataques cibernéticos, como injeções SQL, cross-site scripting (XSS) e outras ameaças comuns. O WAF ajuda a proteger aplicativos web contra vulnerabilidades de segurança.

---

### Web Development

Desenvolvimento web refere-se à criação e manutenção de sites e aplicativos web. Envolve o design, desenvolvimento, teste e implantação de interfaces de usuário e funcionalidades da web.
