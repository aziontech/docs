##  16 de dezembro, 2024

### Data Stream

#### Conector Simple Storage Service (S3)

Agora, ao configurar um conector do tipo Simple Storage Service (S3) para receber dados do Data Stream, os nomes dos arquivos serão compostos por:

- **Object Key Prefix**: o prefixo para o nome do objeto carregado. Exemplo: `user/logs`.
- **Data e hora** em que os pacotes são enviados no formato YYYY/MM/DD/hh/mm/.
- **UUID**: um identificador único de requisição.

Exemplo: `user/logs/2024/10/12/06/24/37d66e78-c308-4006-9d4d-1c013ed89276`.

Essa estrutura simplifica a gestão e monitoramento de logs, permitindo buscas e filtragens precisas usando a data, hora e UUID.

Saiba mais sobre o [Data Stream](/pt-br/documentacao/produtos/observe/data-stream/) e como configurar conectores do [Edge Storage](/pt-br/documentacao/produtos/guias/conector-azion-edge-storage/) e [Amazon S3](/pt-br/documentacao/produtos/guias/amazon-s3-endpoint/) para receber dados.

## 30 de agosto, 2024 

### Políticas de conta

#### Política de Acesso Condicional por Endereço IP

<Tag severity="info" client:only="vue">Preview</Tag>

Agora você pode criar uma política de **Conditional Access by IP Address**, permitindo o acesso aos seus recursos com base em endereços IP específicos, garantindo que apenas dispositivos acessando a partir desses endereços possam alcançar a plataforma.

Consulte o [guia de configuração](/pt-br/documentacao/produtos/guias/conditional-access-by-ip-address/) para mais detalhes.

#### Política de Tempo Limite de Sessão do Usuário

<Tag severity="info" client:only="vue">Preview</Tag>

A política de **User Session Timeout** permite que você defina um **tempo máximo de inatividade** (*maximum idle time*), garantindo que as sessões sejam automaticamente encerradas após um período de inatividade. Ela também permite que você defina um **tempo máximo de sessão** (*maximum session time*) para evitar que as sessões permaneçam abertas indefinidamente.

Consulte a [referência para saber mais](/pt-br/documentacao/produtos/gestao-de-contas/user-session-timeout/) e o guia sobre [como configurá-la](/pt-br/documentacao/produtos/guias/configurar-user-session-timeout/).

#### Política de Bloqueio de Conta

<Tag severity="info" client:only="vue">Preview</Tag>

A política de **Account Lockout** agora está disponível na plataforma da Azion, protegendo contas contra acesso não autorizado. Se um usuário inserir repetidamente a senha errada, excedendo o número máximo permitido de tentativas de login com falha, a conta do usuário será temporariamente bloqueada para evitar novas tentativas.

Leia mais sobre a [Política de Bloqueio de Conta](/pt-br/documentacao/produtos/gestao-de-contas/account-lockout-policy/) e [como configurá-la](/pt-br/documentacao/produtos/guias/configurar-account-lockout-policy/).

### SSO

#### Microsoft Entra como IdP e Integração de Provisionamento Automático de Usuários

Usando **Microsoft Entra Identity Provider (IdP)**, você pode autorizar o single sign-on (SSO) para acessar outros sites ou serviços, como a Azion. Seus usuários podem continuar usando suas identidades corporativas sem precisar lembrar de uma senha específica ou inserir credenciais a cada vez que acessam a plataforma da Azion.

Consulte o guia [Como utilizar Microsoft Entra SAML como um IdP na Azion](/pt-br/documentacao/produtos/guias/sso-microsoft-entra-saml/) para mais informações.

Além disso, com o Microsoft Entra, você pode habilitar o **Automated User Provisioning** para simplificar a gestão de identidade e acesso, criando, atualizando e excluindo automaticamente contas de usuários em aplicativos com base em mudanças no Microsoft Entra ID. Essas mudanças são refletidas nos usuários e equipes da plataforma Azion, uma vez que o recurso esteja habilitado. A automação utiliza o protocolo SCIM, tornando o processo de provisionamento escalável e seguro.

Leia mais sobre este tópico na [documentação](/pt-br/documentacao/produtos/guias/microsoft-entra-provisionamento-automatizado-de-usuarios/).
