**Suporte HTTP/3 para Workloads via IaC (azion.config)**:

**Modernização do sistema de build (tsup → Vite)**:

Como o tsup foi descontinuado, o sistema de build dos pacotes foi modernizado:

- **Build tooling**: Migração de compilação/build de tsup para Vite em todos os pacotes. Adicionados `vite` e `vite-plugin-dts` como dependências de desenvolvimento raiz. Adicionado `vite.config.mjs` por pacote com aliasing interno e configuração customizada.

- **Consistência interna**: Refatorado para usar `import type { ... }` para imports apenas de tipos. Padronizado `export type { ... }` para exports de tipos. Melhorados imports de cliente usando caminhos com alias em vez de imports relativos profundos.

## 11 de julho, 2025

### Bot Manager 1.2.0: Novo engine de fingerprinting

A Azion lançou o **Bot Manager 1.2.0**, com um novo engine de fingerprinting que aprimora a precisão na identificação de dispositivos.

#### Novidade

- **Engine de fingerprinting avançado**: reduz colisões de fingerprinting (diferentes dispositivos recebendo o mesmo fingerprint), melhorando o processo de calibração da sua implementação do Bot Manager.

#### Notas de Implementação

Se você já utiliza o Bot Manager, o código da função foi automaticamente atualizado para a versão 1.2.0. No entanto, para ativar o novo engine de fingerprinting, você precisa atualizar sua configuração de JSON Args:

```json
{
  "engine_version": 2
}
```

:::tip[Importante]
A ativação do novo engine pode causar alterações temporárias nos padrões de classificação de bots. Sua aplicação necessitará de um breve período de adaptação para se ajustar à granularidade aprimorada de fingerprinting.

Até a atualização de seus JSON Args, o Bot Manager continuará utilizando o engine de fingerprinting anterior.
:::

Para detalhes completos de configuração, consulte a [documentação do Bot Manager](/pt-br/documentacao/produtos/secure/edge-firewall/bot-manager/).

## 2 de julho, 2025

### Edge Functions

Azion Edge Functions agora suporta o `WebSocket Proxy`, permitindo que sua Edge Application atue como um servidor ou cliente WebSocket completo e faça proxy de tráfego bidirecional para serviços backend. Isso desbloqueia casos de uso em tempo real, como chat, jogos multiplayer, painéis de telemetria e streams de inferência de AI — todos servidos diretamente no edge para latência de sub-segundos.

**Importante**: Para ativar esse recurso, habilite o módulo Websocket Proxy na Edge Application desejada no Console ou via API. Sem esse pacote, as Cells não podem aceitar ou fazer proxy de conexões WebSocket.

**Visão geral dos recursos**:

- **Modos servidor & cliente**: Use `upgradeWebSocket()` para handlers no lado do servidor ou a API padrão WebSocket(url) para conexões de saída.
- **Helpers de broadcast**: Faça o envio de mensagens para todos os clientes conectados com o mínimo de código.
- **Métricas & logs**: Novos eventos (websocket.connection.accepted, message.sent, message.received) estão disponíveis em Real-Time Metrics e LogStream.
- **Suporte ao Edge Firewall**: O tráfego pode ser inspecionado por regras do Edge Firewall antes da finalização do upgrade.

Para mais detalhes visite a documentação da [WebSocket API](/pt-br/documentacao/runtime/api-reference/websocket/)

#### Compatibilidade com Azion Runtime

Para manter seu código portável e preparado para o futuro, o Azion Runtime agora suporta um conjunto expandido de APIs padronizadas da web. Essas melhorias permitem compressão de streaming nativa, processamento incremental de texto, controle preciso de back-pressure e medições de performance em sub-milissegundos — fornecendo ferramentas avançadas para maximizar o throughput e a observabilidade de cada requisição de Edge Function.

- **Compression Streams API**: `CompressionStream` e `DecompressionStream`
Comprime ou descomprime dados em tempo real (gzip/deflate) com primitivos de streaming em nível de navegador — perfeito para pipelines de logs, respostas de APIs ou transformação de ativos.
- **Streaming Encoding API**: `TextDecoderStream` e `TextEncoderStream`
Decode ou encode UTF-8 (e outros encodings) incrementalmente e com respeito ao back-pressure. Ideal para Server-Sent Events, JSON em chunks e tails de logs em tempo real.
- **Aprimoramento da Streams API**: `CountQueuingStrategy`
Construa streams personalizados com marcas d’água altas baseadas em contagem, mantendo o uso de memória previsível mesmo em workloads com picos.
- **Timer de alta resolução**: `globalThis.performance.now()`
Acesse um relógio monotônico com precisão de sub-milissegundo para medir cold starts, trechos CPU-bound ou latência de ponta a ponta diretamente dentro da sua função.


Essas capacidades alinham o Edge Functions com as últimas especificações [WinterTC](https://wintertc.org/), reforçam a compatibilidade com Node.js, e dão aos desenvolvedores primitivos mais ricos para construir aplicações eficientes, observáveis e em conformidade com padrões.

## 14 de maio, 2025

### Desafio HTTP-01 do Let’s Encrypt via API

A Azion agora oferece emissão e renovação de [certificados Let’s Encrypt](/pt-br/documentacao/produtos/secure/edge-firewall/digital-certificates/) via API usando o desafio HTTP-01, oferecendo um processo simplificado sem a necessidade de registros TXT no DNS. Isso oferece uma solução de integração fácil e conveniente, especialmente benéfica para clientes que gerenciam diversos domínios e nomes de host.

Os certificados são renovados automaticamente antes de expirar, eliminando janelas de manutenção e mantendo as cotas, faturamento e permissões existentes. Perfeito para plataformas de e-commerce e provedores de hospedagem, esse aprimoramento permite um onboarding mais rápido e garante segurança aos sites dos clientes em minutos, sem alterações no DNS.

Para mais informações sobre essas atualizações, visite a documentação de [Digital Certificates](/pt-br/documentacao/produtos/secure/edge-firewall/digital-certificates/).
