## 19 de março, 2025

### Azion Libraries

#### Azion Lib –  SDK JavaScript

Azion apresenta a **Azion Lib**, um SDK abrangente em JavaScript projetado para facilitar interações com diversos serviços da Azion, incluindo **Edge Storage**, **Edge SQL** e funcionalidades de **Purge**. Essa biblioteca versátil é compatível tanto com o **Edge Runtime** da Azion quanto com ambientes externos, garantindo uma integração eficiente entre aplicações diversas.

**Principais recursos**:

- **Interface de cliente unificada**: fornece um cliente centralizado para gerenciar interações com vários serviços da Azion, simplificando os processos de desenvolvimento.

- **Módulos específicos para serviços**: oferece módulos dedicados para **Storage**, **SQL**, **Purge** e outros, permitindo operações mais direcionadas e eficientes.

- **Configuração flexível**: suporta configurações baseadas em variáveis de ambiente e modos de depuração para atender a diversas necessidades de desenvolvimento.

Para instruções detalhadas de instalação e uso, consulte a [documentação do Azion Lib](/pt-br/documentacao/produtos/azion-lib/visao-geral/) ou visite a página do [pacote no NPM](https://www.npmjs.com/package/azion).

O arquivo `azion.config.js`, parte essencial da abordagem de Infrastructure as Code (IaC) da Azion, agora suporta uma gama mais ampla de recursos. Essa melhoria permite configurações mais detalhadas e abrangentes, incluindo:

- **Configurações de WAF**

Para uma visão detalhada das propriedades configuráveis, consulte a [documentação do Azion Config](/pt-br/documentacao/devtools/cli/configs/azion-config-js/).

##  16 de dezembro, 2024

### Data Stream

#### Conector Simple Storage Service (S3)

Agora, ao configurar um conector do tipo Simple Storage Service (S3) para receber dados do Data Stream, os nomes dos arquivos serão compostos por:

- **Object Key Prefix**: o prefixo para o nome do objeto carregado. Exemplo: `user/logs`.
- **Data e hora** em que os pacotes são enviados no formato YYYY/MM/DD/hh/mm/.
- **UUID**: um identificador único de requisição.

Exemplo: `user/logs/2024/10/12/06/24/37d66e78-c308-4006-9d4d-1c013ed89276`.

Essa estrutura simplifica a gestão e monitoramento de logs, permitindo buscas e filtragens precisas usando a data, hora e UUID.

Saiba mais sobre o [Data Stream](/pt-br/documentacao/produtos/observe/data-stream/) e como configurar conectores do [Edge Storage](/pt-br/documentacao/produtos/guias/conector-azion-edge-storage/) e [Amazon S3](/pt-br/documentacao/produtos/guias/amazon-s3-endpoint/) para receber dados.

### Políticas de conta

#### Política de Acesso Condicional por Endereço IP

<Tag severity="info" client:only="vue">Preview</Tag>

Agora você pode criar uma política de **Conditional Access by IP Address**, permitindo o acesso aos seus recursos com base em endereços IP específicos, garantindo que apenas dispositivos acessando a partir desses endereços possam alcançar a plataforma.

Consulte o [guia de configuração](/pt-br/documentacao/produtos/guias/conditional-access-by-ip-address/) para mais detalhes.

#### Política de Tempo Limite de Sessão do Usuário

<Tag severity="info" client:only="vue">Preview</Tag>

A política de **User Session Timeout** permite que você defina um **tempo máximo de inatividade** (*maximum idle time*), garantindo que as sessões sejam automaticamente encerradas após um período de inatividade. Ela também permite que você defina um **tempo máximo de sessão** (*maximum session time*) para evitar que as sessões permaneçam abertas indefinidamente.

Consulte a [referência para saber mais](/pt-br/documentacao/produtos/gestao-de-contas/user-session-timeout/) e o guia sobre [como configurá-la](/pt-br/documentacao/produtos/guias/configurar-user-session-timeout/).

#### Política de Bloqueio de Conta

<Tag severity="info" client:only="vue">Preview</Tag>

A política de **Account Lockout** agora está disponível na plataforma da Azion, protegendo contas contra acesso não autorizado. Se um usuário inserir repetidamente a senha errada, excedendo o número máximo permitido de tentativas de login com falha, a conta do usuário será temporariamente bloqueada para evitar novas tentativas.

Leia mais sobre a [Política de Bloqueio de Conta](/pt-br/documentacao/produtos/gestao-de-contas/account-lockout-policy/) e [como configurá-la](/pt-br/documentacao/produtos/guias/configurar-account-lockout-policy/).

### SSO

#### Microsoft Entra como IdP e Integração de Provisionamento Automático de Usuários

Usando **Microsoft Entra Identity Provider (IdP)**, você pode autorizar o single sign-on (SSO) para acessar outros sites ou serviços, como a Azion. Seus usuários podem continuar usando suas identidades corporativas sem precisar lembrar de uma senha específica ou inserir credenciais a cada vez que acessam a plataforma da Azion.

Consulte o guia [Como utilizar Microsoft Entra SAML como um IdP na Azion](/pt-br/documentacao/produtos/guias/sso-microsoft-entra-saml/) para mais informações.

Além disso, com o Microsoft Entra, você pode habilitar o **Automated User Provisioning** para simplificar a gestão de identidade e acesso, criando, atualizando e excluindo automaticamente contas de usuários em aplicativos com base em mudanças no Microsoft Entra ID. Essas mudanças são refletidas nos usuários e equipes da plataforma Azion, uma vez que o recurso esteja habilitado. A automação utiliza o protocolo SCIM, tornando o processo de provisionamento escalável e seguro.

Leia mais sobre este tópico na [documentação](/pt-br/documentacao/produtos/guias/microsoft-entra-provisionamento-automatizado-de-usuarios/).

## Setembro, 2022

### Updates

* **Edge Firewall**

Otimização de pesquisa por `Domains` e `Last Editor` na tela de **Edge Firewall**.

* **Azion CLI**

- Novas telas de ajuda.
- Nova árvore de comandos para `init`, `build` e `publish`.
- Uso de `Args` foi substituído pelo uso de `Flags`.

### Bug Fixes

* **API — Rules Engine — Edge Firewall**

Correção do tratamento de erro e mensagem para situações quando a API de **Rules Engine** do **Edge Firewall** recebe dois valores `if` para o campo `conditional`.

* **Rules Engine — Edge Application**

Ajuste de comportamento ao implementar uma *Regex* com Behavior Redirect to 301 ou 302 em um *Edge Rule* no **Rules Engine** do **Edge Application**.

* **Edge Functions**

Ajustado o comportamento do uso da estrutura de dados JSON no campo **Args** em **Edge Functions**.

* **Azion CLI**

Mensagem de token expirado corrigida.

* **Credentials**

- Correção de erro que exibia mensagem de sucesso e de erro simultaneamente após ultrapassar 255 caracteres no campo de `nome`.
- Busca em lista de nomes agora considera caracteres acentuados.
- Tabela mantém seu formato, mesmo ao inserir um nome com muitos caracteres.
- Correção de `Internal Error` ao inserir descrição com mais de 200 caracteres.

* **Edge Services**

- Busca em lista de nomes agora considera caracteres acentuados.
- O nome do **Edge Service** é atualizado mesmo após exceder o limite de caracteres.
- Correção de `Internal Error` ao inserir emojis no nome do **Edge Service**.

* **Edge Nodes**

Busca em lista de nomes agora considera caracteres acentuados.

### New Releases

* **Marketplace**

Adição de sistema de versões nas soluções do **Marketplace**, para que clientes possam decidir se querem, ou não, atualizar uma solução, além de poder escolher qual versão utilizar.

## Julho, 2022

### New Releases

**Marketplace**

- **Azion Marketplace** já está disponível para uso. O Marketplace é um catálogo digital que torna mais fácil encontrar, testar e implementar softwares que rodam no edge em qualquer lugar. Com uma ampla variedade de soluções, clientes Azion podem utilizá-lo para aprimorar, compor ou personalizar suas aplicações. O Marketplace também oferece meios para que fornecedores independentes de software (ISVs), desenvolvedores de software e comunidades abertas distribuam softwares que rodam no edge para uma audiência de milhares de compradores ativos. Saiba mais sobre o Marketplace [aqui](/pt-br/documentacao/produtos/marketplace/).

**Azion CLI**

- **Beta Release:** agora, é possível publicar suas aplicações Javascript, Next.js e Flareact na Azion. Além disso, clientes Azion podem gerenciar suas **Edge Functions** e **Edge Services** através de linhas de comando. Saiba mais sobre a Azion CLI [aqui](/pt-br/documentacao/produtos/azion-cli/visao-geral/).
