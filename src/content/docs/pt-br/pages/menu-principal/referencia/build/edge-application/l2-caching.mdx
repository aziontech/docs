**L2 Caching** é um módulo **Edge ⁠Application** que cria uma camada de cache adicional entre o edge e seus servidores de origem, reduzindo a latência e os custos de infraestrutura.

Ao acessar sua edge application na Azion, o usuário acessa a edge network, onde o conteúdo da aplicação é armazenado em cache. Ao ativar o L2 Caching em Edge Application, você pode:

- Manter cache em uma segunda camada no edge.
- Definir longos períodos de TTL para cache.
- Selecionar a região para manter o cache mais próximo de seus usuários.

:::note
L2 Caching só estará disponível se o serviço for ativado para a sua conta. Para ativar o **L2 Caching**, [contate a equipe de Vendas](https://www.azion.com/en/contact-sales/).
:::

## TTL Recomendado

**L2 ⁠Caching** é especialmente projetado para objetos que podem permanecer no cache por um longo período de tempo. Por isso, você só pode ativá-lo em políticas de cache com um TTL igual ou superior a *3 segundos*, sendo o TTL padrão de *60 segundos*.

## Regiões

Quando você ativa o **L2 Caching** como uma camada extra de cache, os servidores edge dessa segunda camada de cache são hospedados nos EUA (`na-united-states`) por padrão. 

A Azion também oferece a opção de hospedar os servidores L2 no Brasil (`sa-brazil`). Para alternar entre as regiões em suas edge applications, entre em contato com a [equipe de Vendas](https://www.azion.com/pt-br/contate-vendas/).

---

## Real-Time Purge

Sempre que necessário, você pode usar o **Real-Time Purge** para expirar seu conteúdo de L2 Caching antes do TTL esperado. Lembre de expirar primeiro na camada L2 e somente depois no **Edge Caching**. Isso evita edge feedback com conteúdo desatualizado em L2.

## Limites

### Bypass cache

O módulo L2 Caching atualmente não suporta bypass cache. Você pode configurar uma regra de [Bypass Cache](/pt-br/documentacao/produtos/guias/edge-caching/#bypass-cache) para armazenamento em cache no edge, mas edge applications com L2 caching ativo continuarão armazenando em cache nessa camada para o TTL especificado.
