O **Functions** para **Applications** permite que você instancie funções serverless em sua application da Azion, configurando também as condições de execução. Essas funções são processadas no edge, respondendo a eventos com velocidade e garantindo escalabilidade para sua aplicação.

> Functions também está disponível para [Firewall](/pt-br/documentacao/produtos/secure/firewall/functions-instances/).

## Instância de função

Para instanciar uma function em uma application, você deve primeiro criar uma função usando **Functions**. Depois que a função for salva, você poderá prosseguir com a instanciação para suas aplicações.

Durante o processo de instanciação, o código da função não pode ser modificado. Em vez disso, os argumentos, ou **Args**, que serão passados para o contexto da execução da função devem ser informados no formato JSON.

## Executar função

Você pode criar uma regra usando [Rules Engine](/pt-br/documentacao/produtos/build/applications/rules-engine/) e, com o comportamento **Run Function**, selecionar a fase de execução e os critérios para que ela aconteca. Quando uma requisição atender aos critérios definidos na regra, a function será chamada e executada diretamente nos edge nodes da Azion.

<LinkButton link="/pt-br/documentacao/produtos/build/applications/rules-engine/#run-function" label="Saiba mais sobre o comportamento Run Function" />

Para obter mais informações sobre a funcionalidade e a aplicação de Functions, explore os [guias de uso](/pt-br/documentacao/produtos/guias/).

## Limites

O campo **Args** suporta um máximo de 100 KB de argumentos.
