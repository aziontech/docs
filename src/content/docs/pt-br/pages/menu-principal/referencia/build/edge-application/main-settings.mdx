## Módulos

A ativação dos módulos listados abaixo pode gerar custos relacionados ao uso. Consulte a [página de preços](https://www.azion.com/pt-br/precos/) para mais informações.

### Application Accelerator

O módulo **Application Accelerator** acelera aplicações web e APIs através de otimizações de protocolo e gerencia requisitos de conteúdo dinâmico.

<LinkButton link="  " label="Saiba mais sobre Application Accelerator" />

### Edge Cache

O **Edge Cache** permite que você configure como sua aplicação faz cache do conteúdo, reduzindo a latência e aumentando as taxas de transferência através da rede de edge nodes da Azion. Isso não apenas melhora o desempenho e escalabilidade do seu conteúdo, mas também reduz os custos de infraestrutura.

<LinkButton link="/pt-br/documentacao/produtos/build/edge-application/edge-cache/" label="Saiba mais sobre Edge Cache" />

### Edge Functions

As **Edge Functions** permitem que você execute código serverless personalizado, mais próximo dos seus usuários. Isso possibilita execução de baixa latência para tarefas como personalização de conteúdo, processamento de requisições de API, melhoria de segurança e integração com serviços de terceiros. Você pode usá-las para criar aplicações dinâmicas, escaláveis e eficientes sem gerenciar infraestrutura de servidor tradicional.

<LinkButton link="/pt-br/documentacao/produtos/build/edge-application/edge-functions/" label="Saiba mais sobre Edge Functions" />

### Image Processor

O **Image Processor** pode manipular e processar imagens diretamente no edge, o que permite otimizar os tempos de transferência de conteúdo e elevar a experiência do usuário em suas aplicações.

<LinkButton link="/pt-br/documentacao/produtos/build/edge-application/image-processor/" label="Saiba mais sobre Image Processor" />

### Tiered Cache

O **Tiered Cache** cria uma camada adicional de cache entre o edge e seus servidores de origem, reduzindo a latência e os custos de infraestrutura.

<LinkButton link="/pt-br/documentacao/produtos/build/edge-application/tiered-cache/" label="Saiba mais sobre Tiered Cache" />

## Debug Rules

Registra regras executadas criadas no Rules Engine. Consulte logs usando Data Stream, Real-Time Events ou Real-Time Events GraphQL API.

Regras que foram executadas com sucesso serão mostradas no campo `$traceback` no Data Streaming e Real-Time Events ou na variável `$stacktrace` no GraphQL.
