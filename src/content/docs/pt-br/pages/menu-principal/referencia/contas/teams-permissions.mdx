Na página **Team Permissions** do Real-Time Manager (RTM), você pode criar e gerenciar equipes, controlar e configurar direitos de acesso para equipes distintas em sua empresa. Você pode, por exemplo, criar grupos dividindo-os de acordo com as tarefas que eles precisam executar no RTM.

Somente usuários com status de **Account Owner** podem adicionar times e editar permissões.

Ao conceder ou restringir o acesso a recursos e configurações específicos, você pode:

- Economizar tempo, pois as equipes acessam apenas o que precisam.
- Aumentar a segurança, limitando o acesso para manter as informações confidenciais seguras.
- Ter mais controle, acompanhando as ações específicas das equipes para questões de segurança e regras.
- Incentivar o trabalho em equipe com o time acessando as ferramentas em conjunto.

## Permissões de acesso

Nesta tabela, você encontra o que cada permissão concede acesso:

| Acesso | Permissão |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| View Content Delivery Settings | Concede permissão para visualizar cache settings list. Permissão para os produtos Web Performance, Real Time Purge, Analytics e Analytics Standard. |
| Edit Content Delivery Settings | Concede permissão para editar a cache settings list. Requer a permissão **View Content Delivery Settings**. Permissão para os produtos Web Performance, Real Time Purge, Analytics e Analytics Standard.|
| Real-Time Purge | Concede permissão para adicionar um novo purge ao Real-Time Purge. |
| View Real-Time Analytics | Concede permissão para visualizar e gerar relatórios no Real-Time Analytics. *Produto descontinuado*. |
| View Security Settings | Concede permissão para visualizar o WAF Rule Sets e Digital Certificates (com exceção da private key), mas não permite criação, remoção ou modificações. |
| Edit Security Settings | Concede permissão para visualizar, criar, editar e remover WAF Rule Sets e Digital Certificates. Também requer a permissão **View Security Settings**. |
| View Subscriptions | Concede permissão para visualizar as assinaturas da conta, mas não permite modificações. Também requer a permissão **View Bills**. |
| Edit Subscriptions | Concede permissão para modificar as assinaturas da conta. Também requer as permissões **View Bills** e **View Subscriptions**. |
| View Payment Methods | Concede permissão para visualizar as formas de pagamento ativas e o histórico de pagamentos. Também requer a permissão **View Bills**. |
| Edit Payment Methods | Concede permissão para criar, editar e remover métodos de pagamento. Também requer as permissões **View Bills** e **View Payment Methods**. |
| View Bills | Concede permissão para visualizar a aba **Billing** em **Billing & Subscriptions**. A visualização de **Payment Methods** e **Payment History** requer a permissão **View Payment Methods**. |
| Edit Multi-Factor Authentication | Concede permissão para editar a opção **Multi-Factor Authentication** na página **Account Settings**. |
| View Edge Applications | Concede permissão para visualizar as configurações, mas não permite modificações, criação ou remoção. |
| Edit Edge Applications | Concede permissão para editar, criar e remover edge applications. Também requer a permissão **View Edge Applications**. |
| View Domains | Concede permissão para visualizar domínios existentes, mas não permite criação, remoção ou modificações. |
| Edit Domains | Concede permissão para criar, editar e remover domínios. Também requer a permissão **View Domains**. |
| View Real-Time Events | Concede permissão para visualizar e filtrar por data source. |
| View Users | Concede permissão para visualizar a conta dos usuários na página **Users Management**, mas não permite criação, remoção ou modificações. |
| Edit Users | Concede permissão para criar, editar e remover usuários da conta. Também requer a permissão **View Users**. |
| Wildcard Purge | Concede permissão para selecionar o purge tipo de **Wildcard** no **Real Time-Purge **. Também requer a permissão **Real Time-Purge**. |
| View Intelligent DNS | Concede permissão para visualizar as zonas criadas na conta. Não permite criação ou remoção. |
| Edit Intelligent DNS | Concede permissão para criar, editar e remover zonas. Também requer a permissão **View Intelligent DNS**. |
| View Data Streaming | Concede permissão para visualizar os streamings da conta. Não permite criação, modificação ou remoção. |
| Edit Data Streaming | Concede permissão para criar, editar e remover zonas. Também requer a permissão **View Data Streaming**. |
| View Network Lists | Concede permissão para visualizar Network Lists, mas não permite criação, remoção ou modificação. |
| Edit Network Lists | Concede permissão para visualizar, criar, editar e remover Listas de Rede. Também requer a permissão **View Network Lists**. |
| View Edge Firewall | Concede permissão para visualizar Edge Firewall Rule Sets, mas não permite criação, remoção ou modificação. |
| Edit Edge Firewall | Ele concede permissão para visualizar, criar, editar e remover conjuntos de regras do Edge Firewall. Também requer a permissão **View Edge Firewall**. |
| View Edge Functions | Concede permissão para visualizar domínios existentes, mas não permite criação, remoção ou modificações. |
| Edit Edge Functions | Concede permissão para criar, editar e remover edge functions. Também requer a permissão **View Edge Functions**. |
| View client list | Concede permissão para visualizar a lista de clientes. Não permite a criação, modificação ou remoção de clientes. O recurso Clients está disponível apenas para contas do tipo Grupo. |
| Change clients | Concede permissão para criar, editar ou remover clientes. Também requer a permissão **View Client list**. |

---

## Tratamento de erros

Se um usuário tentar acessar uma página no RTM sem as permissões necessárias, encontrará uma página de erro *403 Forbidden*.
