O **Object Storage** da Azion é um serviço de armazenamento escalável e seguro projetado para integrar o armazenamento de objetos com a Plataforma de Edge Computing da Azion, utilizando o padrão S3. Os buckets podem ser usados como origens para edge applications ou como diretórios para upload de objetos em tempo real.

Junto com a criação de buckets, você possui controle total sobre a alocação de armazenamento. Isso envolve a gestão de permissões de leitura/escrita e a capacidade de fazer upload, alterar e excluir objetos.

:::note
Essa funcionalidade está disponível via [Azion API](https://api.azion.com), [Edge Runtime](/pt-br/documentacao/produtos/edge-application/edge-functions/runtime/visao-geral/), e [Azion CLI](/pt-br/documentacao/produtos/azion-cli/visao-geral/).
:::

## Buckets

Os buckets são o sistema utilizado para organizar objetos, ou arquivos, armazenados. Semelhantes a pastas, os buckets são contêineres de nível superior para armazenar objetos. Todos os buckets criados com o **Object Storage** da Azion são armazenados no servidor de nuvem *us-west*.

Como os nomes dos buckets identificam os buckets, cada bucket deve ter um nome único. O nome pode conter caracteres alfanuméricos e o traço (`-`). As melhores práticas para nomear buckets incluem especificar que tipos de objetos são armazenados e o tipo de permissões para os objetos. Por exemplo, um bucket para uma edge application `Banking App` em modo somente leitura (read-only) poderia ser nomeado `banking-app-ro`.

---

## Permissões

Ações envolvendo buckets, como fazer upload, listar e excluir objetos, sempre exigem autorização. Além da autenticação necessária pela conta da Azion, essas ações são geridas pelas **Permissions**, ou permissões, do bucket.

As permissões disponíveis são:

- **Somente leitura**: objetos no bucket podem ser lidos, mas não modificados.
- **Leitura e escrita**: objetos no bucket podem ser modificados.

Informações pessoais e sensíveis não devem ser enviadas para o bucket nem incluídas nos nomes de objetos, uma vez que os buckets S3 não atendem aos requisitos de proteção de privacidade.

---

## Origem

O **Object Storage** pode ser usado como uma [origem](/pt-br/documentacao/produtos/edge-application/origins/#object-storage) na **Edge Application** da Azion para recuperar o conteúdo de uma edge application a partir de um bucket.

Você pode determinar se o conteúdo é retirado da raiz do bucket ou de um **Prefix**, ou caminho, dentro do bucket.
