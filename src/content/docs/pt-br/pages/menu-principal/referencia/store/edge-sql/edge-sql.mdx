**Edge SQL** é um produto da Store que permite criar e gerenciar serviços de banco de dados distribuídos com baixa latência. Totalmente compatível com ACID, o Edge SQL é construído sobre um banco de dados alimentado pelo libSQL e funciona enquanto é replicado em nós de borda da Azion. Um banco de dados criado no Edge SQL é armazenado em dois tipos de instâncias: uma instância **principal** e suas **réplicas**.

Com o Edge SQL, você pode:

- Armazenar dados de telemetria em tempo real, ideal para IoT.
- Gerenciar inventário de centros de distribuição de comércio eletrônico/retalho distribuídos.
- Armazenar e analisar registros de acesso, identificando e mitigando rapidamente ataques de segurança.
- Armazenar e gerenciar dados de acesso.

Existem duas maneiras de construir um banco de dados através do Edge SQL: usando a API REST do Edge SQL ou escrevendo uma função de borda. A API REST permite criar, ler, atualizar, excluir e listar bancos de dados. Uma função de borda permite escrever em um banco de dados.

## Regras de negócios

Escrever bancos de dados, através da API REST ou função de borda, só é possível na [instância principal](#instancia-principal). Todas as [réplicas](#replicas) são somente leitura.

:::caution[atenção]
Uma vez que você exclui uma instância principal, essa ação não pode ser desfeita. Você ainda poderá ler as réplicas, mas não poderá escrever nelas.
:::

Durante a criação de um banco de dados, especialmente o primeiro, há um tempo de propagação para que ele seja criado nos nós de borda. A resposta da sua solicitação `POST` exibirá um estado de `criando` e você pode executar solicitações `GET` para verificar quando estiver criado.

Você também pode usar uma função de borda JavaScript para consultar e analisar se os snapshots foram mesclados em réplicas.

## Resiliência de dados

Com uma arquitetura distribuída em execução nos nós de borda da Azion, o Edge SQL garante que os dados sejam mantidos seguros e disponíveis. Seus dados são criados em uma instância principal e propagados por réplicas nos edge nodes da Azion.

## Bancos de dados

Um banco de dados é um conjunto de tabelas estruturadas que permitem escrever e ler dados na linguagem SQL. Organizados em colunas e linhas, os usuários podem adicionar dados logicamente, com colunas representando informações específicas no eixo vertical, enquanto as linhas representam dados de entrada individuais, relacionados a uma coluna, no eixo horizontal.

[Criar um banco de dados](guidelink) é o primeiro passo para usar o Edge SQL e adicionar dados:

```bash
curl --location 'https://stage-edge-sql-api.azion.net/edge-sql/api/databases' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
   "name": "meu-banco-de-dados"
}'
```

### Query

Uma query é como você estrutura sua tabela ou uma busca em linhas em um banco de dados previamente criado. É possível fazer consultas com base em relações e restrições.

Por exemplo, para criar uma tabela via cURL:

```bash
curl --location 'https://stage-edge-sql-api.azion.net/edge-sql/api/databases/{id_banco_de_dados}/consulta' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
    "statements": [
        "CREATE TABLE usuarios (id INTEGER PRIMARY KEY, nome TEXT NOT NULL);"
    ]
}'
```

Para consultar dados na tabela já criada:

```bash
curl --location 'https://stage-edge-sql-api.azion.net/edge-sql/api/databases/{id_banco_de_dados}/consulta' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
    "statements": [
        "SELECT * FROM usuarios;"
    ]
}'
```

## Instâncias

### Instância principal

A instância de armazenamento primária, ela recebe todos os comandos de *escrita* e sincroniza os dados recebidos com as réplicas.

Todos os bancos de dados](#bancos-de-dados) são criados e todas as [consultas](#consulta) são escritas na instância principal. Após o processo automático gerenciado pela Azion, seus dados são processados em edge locations, snapshots são criados e mergeados em réplicas.

:::caution[atenção]
É recomendado sempre criar um backup da sua [instância principal](#instancia-principal) para garantir que seus dados estejam seguros. Como há um tempo de propagação que difere para as réplicas, é melhor fazer backup da principal do que das réplicas.
:::

### Réplicas

Réplicas são cópias *somente leitura* da instância principal. As réplicas são atualizadas em tempos de propagação diferentes, então eventualmente estarão sincronizadas na instância principal, mas isso não ocorre instantaneamente.

O processo de replicação ocorre automaticamente através de sistemas de backend da Azion, nos quais snapshots são criados e então mergeados em cada réplica.

Elas lidam com armazenamento, processamento e gerenciamento de dados próximos aos dispositivos que leem os dados, auxiliando na distribuição, disponibilidade e baixa latência de dados.

## Linguagens de consulta suportadas

Como o **Edge SQL** da Azion é baseado no SQLite, ele suporta as mesmas linguagens SQL que o SQLite.

[Veja as linguagens SQL entendidas](https://www.sqlite.org/lang.html) para criar queries.

## Limites

**Aumente limites** <br></br>
Você pode solicitar o aumento dos limites com base no seu plano. Contate o [time de suporte técnico](/pt-br/documentacao/servicos/suporte/) para fazer a solicitação.

Estes são os **limites default**:

| Escopo | Limite | 
| ----- | ----- | 
| Número máximo de colunas por tabela | 100 |
| Duração máxima da consulta SQL | 30 segundos |

| Escopo | Developer | Business | Enterprise | Mission Critical |
| ------ | --------- | -------- | ---------- | ---------------- |

| Tamanho máximo do banco de dados | 200 MB | 500 MB | 2 GB | 2 GB |
