Para instalar e implantar templates ou integrações da Azion, pode ser necessário conceder privilégios ao **Marketplace** para ler e gravar as configurações de suas edge applications.

---

## Permissões comuns para integrações

Toda integração tem suas próprias necessidades de privilégios. Por exemplo, uma integração pode precisar de permissão para ler e escrever no **Rules Engine** da sua edge application, a fim de criar uma regra para acionar a execução da integração.

Esse guia detalha as permissões comuns que você deve autorizar no Marketplace no momento de instalar integrações.

### Main Settings

As **Main Settings** da sua edge application incluem as principais configurações necessárias para executar suas integrações. Quando uma integração solicita privilégios para gravar e ler as **Main Settings** de uma edge application, isso significa que essa integração precisa ler e/ou gravar uma das seguintes configurações:

- **Functions**
   - Deve estar habilitada para executar qualquer integração do **Marketplace**. Para fazer isso, é obrigatório conceder tais privilégios se você quiser executar qualquer integração instalada via **Marketplace**. Essa opção é usada para alterar a requisição/resposta, personalizar argumentos (**Args**) para suas funções e escolher *critérios* de execução no **Rules Engine**.
- **Application Acceleration**
   - Essa opção será obrigatória se a sua solução precisar usar regras avançadas dentro do **Rules Engine**, tais como `Bypass⁠Cache` ou `Forward Cookies`.
- **Image Processor**
   - Ajuda a otimizar a experiência com imagens em sua edge application.
- **Load Balancer**
   - Consiste em um balanceador de tráfego para garantir a melhor confiabilidade e controle de congestionamento de rede.

:::note
Algumas integrações no Marketplace da Azion podem exigir o módulo de cache L2. Entre em contato com a [equipe de vendas da Azion](/pt-br/contate-vendas/) para ativar este módulo para sua conta.
:::

### Rules Engine

Cada edge application depende do **Rules Engine** para executar os *behaviors* (comportamentos) e *criteria* (critérios) necessários para executar a sua edge application. Para isso, o **Marketplace da Azion** precisa de privilégios para ler e escrever suas regras no Rules Engine.

### Edge Firewall

Algumas integrações são executadas na instância do **Edge Firewall**. Essas integrações precisam de mais privilégios do que os concedidos a uma edge application. Para conhecer os privilégios de acesso do Edge Firewall, visite a documentação de [Teams Permissions](/pt-br/documentacao/produtos/gestao-de-contas/teams-permissions/).

## Requisitos comuns para templates

Para garantir o desempenho ideal de certos templates, alguns produtos da Azion devem ser ativados em sua conta e você deve ativá-los separadamente via RTM. Se esses produtos forem ativados, a execução do template poderá gerar custos relacionados ao uso. Consulte os [guias de cada template](/pt-br/documentacao/produtos/guias/#azion-templates) e a [página de preços](/pt-br/precos/) para obter mais informações.

Alguns templates funcionam com ferramentas de terceiros para fornecer uma opção mais abrangente e moderna. O uso desse tipo de templates exigirá a conformidade com certos requisitos, como a criação de uma conta ou a configuração inicial. Em alguns casos, essa integração pode gerar custo por uso.
