Este guia fornece um passo a passo para migrar aplicações existentes para a Azion. Ele descreve os requisitos necessários e detalha os passos para uma transferência bem-sucedida de aplicações de outros serviços para uma arquitetura distribuída.

---

## Migrando de outras plataformas

Se você está migrando de uma plataforma específica, a Azion oferece guias de migração dedicados:

- **AWS**: [Guia de migração da AWS para a Azion](/pt-br/documentacao/produtos/guias/aws-guia-migracao/) - cobre CloudFront, Lambda, S3, DynamoDB, Route 53, WAF, proteção DDoS e observabilidade
- **Cloudflare**: [Guia de migração da Cloudflare para a Azion](/pt-br/documentacao/produtos/guias/cloudflare-guia-migracao/) — cobre Workers, KV, R2, D1, DNS, WAF e mais
- **Fastly**: [Guia de migração da Fastly para a Azion](/pt-br/documentacao/produtos/guias/fastly-guia-migracao/) - cobre Full-Site Delivery, Compute, Edge Data Storage, Object Storage, WAF, logging e observabilidade

## Por que migrar para a Azion

Migrar para a Azion Web Platform oferece vantagens distintas para aplicações modernas:

**Performance**
- Execução sem cold starts em uma arquitetura distribuída
- Deploys instantâneos com atualizações sem downtime
- Pontos de presença globais que executam lógica próxima aos seus usuários

**Eficiência de custos**
- Zero taxas de egresso para transferência de dados pela infraestrutura da Azion
- Preços previsíveis sem cobranças ocultas de bandwidth

**Unificação de plataforma**
- Build, secure, store e observe em um ambiente integrado
- APIs e padrões de configuração consistentes em todas as capacidades
- Plataforma única para aplicações, functions, storage, databases e segurança

:::note
Se sua conta está sob um plano de serviço **Business** ou superior, você tem acesso a especialistas por meio dos [Integration Services](/pt-br/documentacao/servicos/solutions-lab/integration-services/). Para mais informações detalhadas, veja os [Planos de Serviço da Azion](https://www.azion.com/pt-br/servicos-profissionais/).
:::

## Pré-requisitos

Antes de iniciar a migração, certifique-se de ter o seguinte:

- **Conta Azion**: Crie uma conta gratuita no [Azion Console](https://console.azion.com/). Você precisará de acesso ao Console para criar aplicações e configurar domínios.
  <LinkButton link="/pt-br/documentacao/produtos/guias/criar-conta-azion/" label="Criar uma conta Azion" severity="secondary" target="_blank" />

- **Azion CLI** (opcional, mas recomendado): Instale a CLI para desenvolvimento local e deploy. A CLI oferece um fluxo de trabalho simplificado para construir, testar e implantar aplicações.
  <LinkButton link="/pt-br/documentacao/produtos/azion-cli/visao-geral/" label="Visão geral da Azion CLI" severity="secondary" target="_blank" />

## Antes de começar

Antes de migrar aplicações para a Azion, existem duas variáveis principais que devem existir:

1. **Connector**: o servidor de origem da aplicação. Este ponto de entrada assume a forma de um host, um endereço IPv4 ou IPv6 de um serviço de armazenamento ou uma aplicação existente.
  <LinkButton link="/pt-br/documentacao/produtos/secure/connectors/" label="Ir para referência de Connectors" severity="secondary" target="_blank" />

2. **Workload**: o domínio alvo da aplicação. Esta saída assume a forma de um domínio Azion, que pode ser apontado para um domínio personalizado.
  <LinkButton link="/pt-br/documentacao/produtos/secure/workloads/" label="saiba mais sobre Workloads" severity="secondary" target="_blank" />

A maioria das aplicações precisa de um connector do qual a Azion recupera o conteúdo e o código que as compõem. Essa aplicação passa pelos pontos de presença globais da Azion para ser entregue aos usuários através de um domínio associado a um workload.

<LinkButton link="/pt-br/documentacao/produtos/guias/build/visao-geral/" label="saiba mais sobre applications" severity="secondary" target="_blank" />

## Passo 1: Crie uma application

Você pode começar criando uma application usando um template básico. Uma opção recomendada é o template [Hello World](/pt-br/documentacao/produtos/guias/hello-world-template/), que usa **Functions** para executar uma função simples em uma arquitetura distribuída.

<LinkButton link="/pt-br/documentacao/produtos/marketplace/visao-geral-templates-e-integracoes/" label="Ir para referência de Templates" severity="secondary" target="_blank" />

Veja a [lista completa de templates disponíveis](/pt-br/documentacao/produtos/guias/#azion-templates).

---

## Passo 2: Teste localmente

Antes de implantar a aplicação, a recomendação é [fazer o staging da aplicação através do arquivo hosts](/pt-br/documentacao/produtos/guias/testar-edge-application-atraves-do-arquivo-hosts/) para garantir que tudo está funcionando corretamente. Este passo ajuda a identificar e resolver quaisquer erros de configuração ou problemas causados pela migração.

---

## Passo 3: Aponte o domínio para um workload

O passo final para migrar totalmente sua aplicação para a Azion é apontar seu domínio personalizado para o workload da Azion. Ao fazer isso, você permite que os usuários acessem sua application através de seu domínio, reduzindo a latência e os custos de processamento na origem.

<LinkButton link="/pt-br/documentacao/produtos/guias/apontar-dominio-para-a-azion/" label="consulte o guia de como apontar um domínio para um workload" />

## Após a migração

Depois que sua aplicação estiver rodando na Azion, configure estas funcionalidades essenciais:

### Configure observabilidade

Monitore a performance e o comportamento da sua aplicação em tempo real:

- **[Real-Time Metrics](/pt-br/documentacao/produtos/observe/real-time-metrics/)** — Visualize requisições, bandwidth, latência e taxas de cache hit através de dashboards e GraphQL API
- **[Real-Time Events](/pt-br/documentacao/produtos/observe/real-time-events/)** — Acesse logs detalhados de requisições, functions, eventos de WAF e mais para troubleshooting

### Habilite funcionalidades de segurança

Proteja sua aplicação com as capacidades de segurança da Azion:

- **[Web Application Firewall](/pt-br/documentacao/produtos/secure/firewall/web-application-firewall/)** — Proteção contra SQL injection, XSS e outros ataques de camada de aplicação
- **[DDoS Protection](/pt-br/documentacao/produtos/secure/firewall/ddos-protection/)** — Proteção automática contra ataques volumétricos e de protocolo
