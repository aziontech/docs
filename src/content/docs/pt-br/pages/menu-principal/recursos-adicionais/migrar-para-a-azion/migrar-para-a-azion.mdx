Este guia fornece um passo a passo para migrar aplicações existentes para a Azion. Ele descreve os requisitos necessários e detalha os passos para uma transferência bem-sucedida de aplicações de outros serviços para o edge.

:::note
Se sua conta está sob um plano de serviço **Business** ou superior, você tem acesso a especialistas por meio dos [Integration Services](/pt-br/documentacao/servicos/solutions-lab/integration-services/). Para mais informações detalhadas, veja os [Planos de Serviço da Azion](https://www.azion.com/pt-br/servicos-profissionais/).
:::

---

## Antes de começar

Antes de migrar aplicações para a Azion, existem duas variáveis principais que devem existir:

1. **Endereço de origem**: o servidor de origem da aplicação. Esta entrada para o edge assume a forma de um host, um endereço IPv4 ou IPv6 de um serviço de armazenamento ou uma aplicação existente.
  <LinkButton link="/pt-br/documentacao/produtos/build/edge-application/origins/" label="Ir para referência de Origins" severity="secondary" target="_blank" />

2. **Domínio de entrega**: o domínio alvo da aplicação. Esta saída assume a forma de um domínio Azion, que pode ser apontado para um domínio personalizado. 
  <LinkButton link="/pt-br/documentacao/produtos/build/edge-application/domains/" label="saiba mais sobre Domains" severity="secondary" target="_blank" />

A maioria das aplicações precisa de uma origem da qual a Azion recupera o conteúdo e o código que as compõem. Essa aplicação passa pelos edge nodes da Azion para ser entregue aos usuários através de um domínio.

<LinkButton link="/pt-br/documentacao/produtos/guias/build/visao-geral/" label="saiba mais sobre edge applications" severity="secondary" target="_blank" />

## Passo 1: Crie uma edge application

Você pode começar criando uma application usando um template básico e o endereço de origem de sua aplicação. Uma opção recomendada é o template [Hello World](/pt-br/documentacao/produtos/guias/hello-world-template/), que usa **Functions** para executar uma função simples no edge.

<LinkButton link="/pt-br/documentacao/produtos/marketplace/visao-geral-templates-e-integracoes/" label="Ir para referência de Templates" severity="secondary" target="_blank" />

Veja a [lista completa de templates disponíveis](/pt-br/documentacao/produtos/guias/#azion-templates).

---

## Passo 2: Teste localmente

Antes de implantar a aplicação, a recomendação é [fazer o staging da aplicação através do arquivo hosts](/pt-br/documentacao/produtos/guias/testar-edge-application-atraves-do-arquivo-hosts/) para garantir que tudo está funcionando corretamente. Este passo ajuda a identificar e resolver quaisquer erros de configuração ou problemas causados pela migração.

---

## Passo 3: Aponte o domínio

O passo final para migrar totalmente sua aplicação para a Azion é apontar seu domínio personalizado para a edge application da Azion. Ao fazer isso, você permite que os usuários acessem sua edge application através de seu domínio, reduzindo a latência e os custos de processamento na origem.
