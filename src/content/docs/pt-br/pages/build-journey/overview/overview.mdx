Os produtos Azion Build são o seu ponto de partida para utilizar o poder do edge computing em suas aplicações.

## Edge Application

**Edge Application** permite que você construa aplicações mais próximas dos usuários finais, oferecendo processamento de baixa latência no edge. Quando você constrói uma edge application, você pode determinar a origem do seu conteúdo e configurar o domínio que será usado para entregar sua edge application aos usuários finais.

<Button href="/pt-br/documentacao/produtos/edge-application/" variant="secondary" text="Ir para a documentação da Aplicação Edge"></Button>

Através da arquitetura de proxy reverso, a edge processa as solicitações do usuário e mantém os dados em cache, descentralizando o armazenamento de objetos e otimizando as taxas de transferência de dados e eficiência. Esta escalabilidade permite que você se adapte ao crescimento da demanda de tráfego sem sobrecarregar sua infraestrutura e reduzindo os custos de manutenção.

![Visão geral de uma solicitação/resposta através da Edge Application](/assets/docs/images/uploads/edge-app-overview.png)

Quando um usuário solicita conteúdo de sua edge application, o navegador converte o domínio solicitado em um endereço IP pré-definido via resolução DNS. Isso direciona a solicitação para um edge node, onde todo o processamento de uma aplicação ocorre. Nesta etapa, além das personalizações de segurança e observabilidade, o edge node processa e inclui quaisquer modificações de segurança e observabilidade. A solicitação então passa por cada configuração em Edge Application, e qualquer conteúdo em cache na edge será entregue diretamente na resposta. Após isso, o sistema usa algoritmos de load balancing distribui solicitações entre múltiplas origens. Após a mediação pela Plataforma Azion, a solicitação chega à sua origem, que irá responder à requisição. A edge processa novamente esta resposta antes de entregá-la ao usuário.
