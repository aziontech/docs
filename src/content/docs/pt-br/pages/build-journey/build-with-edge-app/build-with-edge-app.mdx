[Edge Application](/pt-br/documentacao/produtos/edge-application/) permite que você configure várias configurações, desde comportamentos de requisição-resposta até como o edge armazena seu cache. Este guia ajudará você a começar a construir uma edge application do zero.

:::tip
Tem uma aplicação e quer mover para o edge? [Migre para Azion](/pt-br/documentacao/produtos/guides/migrar-ns-para-a-azion/). Já trabalha com um determinado framework ou tem uma infraestrutura em mente? [Verifique os Templates disponíveis](/pt-br/documentacao/produtos/link_templates).
:::

## Via RTM

1. [Acesse RTM](https://manager.azion.com).
2. No canto superior esquerdo da página, abra o **Products menu**, representado por três linhas horizontais, e selecione **Edge Application**.
3. Clique no botão **Add Edge Application**.
4. Dê um nome à sua aplicação.
5. Em **Delivery Protocol**, selecione **HTTP**.
6. Em **HTTP Ports**, mantenha a opção **80 (Default)** selecionada.
7. Em **Origin Type**, selecione **Single Origin**.
8. Em **Address**, adicione seu endereço de origem no formato Host ou IPv4. Por exemplo: `httpbin.org`.
9. Em **Origin Protocol Policy**, selecione a opção **Preserve HTTP/HTTPS protocol**.
10. Em **Host Header**, selecione a variável `${host}`.
11. Em **Browser Cache Settings**, mantenha a opção **Honor Origin Cache Headers** selecionada.
12. Em **CDN Cache Settings**, mantenha a opção **Honor Origin Cache Settings** selecionada.
13. Em **Default TTL (seconds)**, mantenha o valor `60`.
14. Clique no botão **Save**.

Agora sua aplicação está construída, mas você não pode acessá-la usando um navegador porque precisa lançá-la através de um domínio. Para criar um domínio:

15. Clique no botão **Add Domain**.
16. Dê um nome ao seu domínio.
17. Em **Edge Certificate**, selecione o certificado **Azion SAN**.
18. Em **Edge Application**, selecione sua edge application.
19. Desative o **CNAME Access Only**. Isso permitirá que os usuários acessem sua aplicação usando o endereço `xxxxxxxxxx.map.azionedge.net`.
20. Clique no botão **Save**.

Aguarde alguns minutos para que seu novo domínio se propague para os edge nodes e acesse seu endereço `xxxxxxxxxx.map.azionedge.net`. Agora você deve ver sua edge application online.

## Via CLI

Você também pode criar sua aplicação via interface de linha de comando (CLI). Visite a [documentação CLI](/pt-br/documentacao/produtos/build/develop-with-azion/cli/) para instalar o **Azion CLI** no seu computador.

## Via API

1. Execute a seguinte solicitação `POST` no seu terminal, substituindo `[TOKEN VALUE]` pelo seu [personal token](/pt-br/documentacao/produtos/guides/personal-tokens/):

```bash
curl --location 'https://api.azionapi.net/edge_applications' \
--header 'Accept: application/json; version=3' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
    "name": "httpbin.org",
    "delivery_protocol": "http",
    "http_port": [80],
    "origin_type": "single_origin",
    "address": "httpbin.org",
    "origin_protocol_policy": "preserve",
    "host_header": "${host}",
    "browser_cache_settings": "honor",
    "cdn_cache_settings": "honor",
    "cdn_cache_settings_maximum_ttl": 60
}'
```

2. Você receberá uma resposta confirmando a criação de sua edge application:

```json
{
    "results": {
        "id": <edge_application_id>,
        "name": "httpbin.org",
        "delivery_protocol": "http",
        "http_port": [
            80
        ],
        "https_port": [
            443
        ],
        "minimum_tls_version": "tls_1_3",
        "active": true,
        "debug_rules": false,
        "http3": false,
        "websocket": false,
        "supported_ciphers": "all",
        "application_acceleration": false,
        "caching": true,
        "device_detection": false,
        "edge_firewall": false,
        "edge_functions": false,
        "image_optimization": false,
        "load_balancer": false,
        "raw_logs": false,
        "web_application_firewall": false
    }
}
```

3. Copie o valor do campo `id`.
4. Execute o próximo comando `POST` para criar um domínio, substituindo a variável `<edge_application_id>` pelo `id` de sua edge application recebido na resposta anterior:

```bash
curl --location 'https://api.azionapi.net/domains' \
--header 'Accept: application/json; version=3' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
    "name": "httpbin.org - domain",
    "cname_access_only": false,
    "edge_application_id": <edge_application_id>
}'
```

5. Você receberá uma resposta semelhante a esta:

```json
{
    "results": {
        "id": <domain_id>,
        "name": "httpbin.org - domain",
        "cnames": [],
        "cname_access_only": false,
        "digital_certificate_id": null,
        "edge_application_id": <edge_application_id>,
        "is_active": true,
        "domain_name": "xxxxxxxxxx.map.azionedge.net",
        "environment": "production",
        "is_mtls_enabled": false,
        "mtls_verification": "enforce",
        "mtls_trusted_ca_certificate_id": null,
        "edge_firewall_id": null
    }
}
```

6. Como o `domain_name` recebido na resposta carrega o domínio Azion de sua aplicação como uma string, você precisa esperar alguns minutos para que as alterações se propaguem. Em seguida, cole o valor entre aspas duplas (`"`) no seu navegador para acessar sua aplicação.

:::tip

Além disso, consulte a [documentação da API Azion](https://api.azion.com/) e a [especificação OpenAPI](https://github.com/aziontech/azionapi-openapi/) para saber mais sobre todos os recursos disponíveis via API.
