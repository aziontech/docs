Veja o comando que permite o gerenciamento de [personal tokens](/pt-br/documentacao/produtos/gestao-de-contas/personal-tokens/) por meio da Azion CLI.

---

## personal_token create

### Uso

```sh
azion personal_token create --name "novoToken" --expiration "9m" 
```

### Flags opcionais

#### in 

A opção `--in` informa o caminho do arquivo JSON que contém todos os atributos do personal token que está sendo criado.

**Exemplo**

```sh
{
    "name": "Token de um dia",
    "expires_at": "9m"
}
```

#### help

A opção `--help` exibe mais informações sobre o subcomando `create`.

### Flags obrigatórias quando --in não é informado

#### description 

A opção `--description` informa a descrição do personal token.

#### expiration 

A opção `--expiration` informa a data de expiração do personal token.

**Exemplo** 

```sh 
azion personal_token create --name "xxtoken" --expiration "9m" // meses
azion personal_token create --name "xxtoken" --expiration "9d" // dias
azion personal_token create --name "xxtoken" --expiration "1a" // ano
```

#### name

A opção `--name` informa o nome do personal token.

## personal_token list

### Uso

```sh
azion personal_token list [flags]
```

### Flags opcionais

#### details

A opção `--details` exibe todos os campos relevantes ao listar.

#### help

A opção `--help` exibe mais informações sobre o subcomando `list`.

--- 

## personal_token delete

### Uso

```sh
azion personal_token delete --id xxxxxx-xxxxx-xxxx-xxxxx-xxxx
```

### Flags obrigatórias 

#### id

A opção `--id` informa o identificador único do personal token que está sendo excluído.

#### Flags opcionais

#### help

A opção `--help` exibe mais informações sobre o subcomando `delete`.
