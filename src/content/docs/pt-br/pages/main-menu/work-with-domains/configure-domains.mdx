Com a Azion, você pode criar e executar aplicações diretamente no edge. Para permitir o acesso à sua aplicação através do navegador e começar a receber usuários e monitorar métricas, você precisa de um domínio. Com [Domains](/documentacao/produtos/edge-application/domains/), a Azion fornece um endereço de domínio gratuito no formato `xxxxxxxxxx.map.azionedge.net`. Quando os usuários acessam sua aplicação através deste domínio Azion, o node mais próximo dentro da edge network busca e entrega a edge aplication.

:::tip
Não tem uma edge application? [Construa com Edge Application](link_build_edge_app) ou [use os templates disponíveis](link_templates).
:::

## Via RTM

Quando você cria uma aplicação [usando um template](link_templates) ou [via CLI](link_cli), sua aplicação é automaticamente associada a um domínio Azion.Você pode [configurar um domínio](#configure-seu-dominio) ou criar um novo:

1. [Acesse o RTM](https://manager.azion.com).
2. No canto superior esquerdo da página, abra o **Products menu**, representado por três linhas horizontais, e selecione **Domains**.
3. Clique no botão **Add Domain**.
4. Dê um nome ao seu domínio.
5. Em **Edge Certificate**, selecione o certificado **Azion SAN**.
6. Em **Edge Application**, selecione sua edge application.
7. No campo **CNAME**, você pode listar domínios personalizados para vincular à sua aplicação; por exemplo: `seudominio.com`. Veja a [seção de domínios personalizados](#custom-domains) para saber mais.
8. Desative o **interruptor de acesso apenas por CNAME**. Isso permitirá que os usuários acessem sua aplicação usando o endereço `xxxxxxxxxx.map.azionedge.net`.
9. Clique no botão **Save**.

Aguarde alguns minutos para que seu novo domínio se propague para os edge nodes e acesse o endereço `xxxxxxxxxx.map.azionedge.net` para visualizar sua edge application online.

## Via API

1. Execute a seguinte requisição `POST`, substituindo `[TOKEN VALUE]` pelo seu [personal token](/en/documentacao/products/guides/personal-tokens/), a variável `<edge_application_id>` pelo [ID da sua edge application](/en/documentacao/products/guides/build-with-edge-application/):

```bash
curl --location 'https://api.azionapi.net/domains' \
--header 'Accept: application/json; version=3' \
--header 'Authorization: Token [TOKEN VALUE]'\
--header 'Content-Type: application/json' \
--data '{
    "name": "example.org - domain",
    "cname_access_only": false,
    "digital_certificate_id": null,
    "edge_application_id": <edge_application_id>
}'
```

    | Chave | Descrição |
    | --- | --- |
    | name | Define a string em valor como um nome da entrada de domínio com Azion. Você pode modificar este valor para se adequar às suas necessidades, pois ele não representa um endereço de domínio. |
    | cname_access_only | Quando definido como false, permite o acesso via endereço Azion. |
    | digital_certificate_id | Quando null, seleciona o certificado Azion SAN. Se você quiser associar seu próprio certificado, veja Criar um certificado digital. |
    | edge_application_id | O ID da aplicação de borda vinculada ao domínio. Para obter o ID, veja Construir com Aplicação de Borda |
