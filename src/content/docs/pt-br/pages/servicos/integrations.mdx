**Integration Services** é a forma mais rápida e segura para migrar seus conteúdos, aplicações web e APIs para a Azion, não importando a complexidade e o volume de recursos que você utiliza. O time de Integration Services irá ativar os serviços contratados por você e prover as ferramentas, documentações e treinamentos necessários para seu sucesso utilizando a plataforma da Azion.

| Plano | Horas/Ano | Horas adicionais disponíveis |
| ----- | --------- | ---------------------------- |
| Developer | Não disponível | Não disponível |
| Business | 5 horas | Sim |
| Enterprise | 20 horas | Sim |
| Mission Critical | 60 horas | Sim |

Para obter informações mais detalhadas, consulte as opções de [Planos de Serviço](/pt-br/documentacao/servicos/planos-de-servico/).

Um especialista da Azion estará disponível para apoiar o cliente em todo o processo através de:

1. Ativação dos serviços Azion contratados pelo cliente conforme descritos na *Ordem de Serviço*.
2. Levantamento de requisitos técnicos e de negócios junto ao cliente.
3. Treinamento de uso do Real-Time Manager.
4. Configuração dos serviços da Azion para o cliente.
5. Revisão e otimização de configurações dos serviços Azion de acordo com as demandas do cliente.

O especialista da Azion irá seguir um processo de integração composto pelas seguintes etapas:

| Estágio | Responsabilidades do Cliente | Responsabilidades da Azion |
| ------- | ---------------------------- | -------------------------- |
| 1. Alocação de recursos para o projeto, levantamento de requisitos, treinamento, ferramentas e revisão de documentações. | - Designar os recursos necessários para colaborar com o time de Integration Services.<br/><br />- Revisar documentações técnicas e guias de implementação.<br/><br />- Apoiar internamente no levantamento de requisitos técnicos e de negócio guiado pela Azion, respondendo as perguntas realizadas e fornecendo informações. | - Designar um especialista de Integrations. <br/><br />- Agendar o início do projeto com o Customer Success Manager responsável para a coleta de requisitos.<br/><br />- Realizar uma análise detalhada de necessidades apresentadas pelo cliente, entendendo requisitos técnicos e de negócios a partir das descrições apresentadas pelo cliente ou pela análise do conteúdo ou aplicação. |
| 2. Planejamento do projeto | - Confirmar agenda do projeto, plano de implementação e plano de testes.<br/><br /> - Criar plano e teste funcional e condições para teste de aceitação. | - Identificar e projetar as soluções adequadas para cada requisito técnico ou de negócio do cliente.<br/><br />- Planejar cronograma de implementação e testes.<br/><br />- Validar plano de teste funcional e condições para teste de aceitação do cliente. |
| 3. Implementação | - Preparar a infraestrutura de origem para entregar o conteúdo através da plataforma da Azion. | - Realizar a configuração no Real-Time Manager para o cliente.<br/><br />- Disponibilizar as configurações necessárias para os testes por parte do cliente.<br/><br /> - Validar as configurações de origem executadas pelo cliente para integração com a Azion. |
| 4. Testes | - Testar o conteúdo e aplicação no ambiente disponibilizado pela Azion para garantir o funcionamento.<br/><br />- Obter aceite dos stakeholders para o funcionamento da plataforma, conforme esperado. | - Testar o conteúdo e aplicação para os requisitos levantados durante a fase inicial do projeto.<br/><br />- Testar o desempenho de entrega e a conquista dos objetivos propostos. |
| 5. Onboarding | - Executar a implementação do conteúdo ou aplicação para a plataforma da Azion.<br/><br />- Testar o ambiente de produção, acompanhar gráficos e logs de acesso.<br/><br />- Solicitar rollback em caso de problemas. | - Observar o comportamento do ambiente de produção para identificar eventuais anomalias.<br/><br />- Assistir o cliente no que for necessário durante a virada de produção.<br/><br /> - Acompanhar o processo e os testes do cliente. |
| 6. Conclusão | - Validar o sucesso dos objetivos do projeto.<br/><br />- Comunicar os procedimentos de suporte para o time responsável pela manutenção do conteúdo e da aplicação. | - Agendar reunião de review.<br/><br />- Conduzir, junto ao cliente, revisão e validação do sucesso do projeto.<br/><br />- Analisar as necessidades que não foram levantadas durante o planejamento e fazer ajustes.<br/><br /> - Revisar procedimentos de suporte para incidentes e problemas em produção.<br/><br />- Encerrar projeto. |

:::note[nota]
Todo o processo de integração é conduzido remotamente, por *telefone* ou *videoconferência*.:
:::
