**Orchestrator** permite que você gerencie e controle recursos de edge em tempo real, incluindo a implantação, atualização e gerenciamento de vários serviços e recursos.

A Azion oferece duas maneiras de gerenciar edge nodes, edge services e recursos:

- Via [Azion API](https://api.azion.com/#9a7daab9-63f7-43a9-9959-2bd91088fca8)
- Via [Azion Console](/pt-br/documentacao/produtos/guias/como-acessar-o-azion-console/)

:::note 
A criação de um edge node acontece diretamente nos dispositivos quando o [Orchestrator Agent](/pt-br/documentacao/produtos/guias/deploy/instalar-orchestrator-agent/) é instalado neles.
:::

## Passo 1: Crie um edge service

Para criar um edge service, você precisa dar a ele um nome fácil de lembrar.

<LinkButton link="/pt-br/documentacao/produtos/guias/deploy/criar-edge-service/" label="consulte o guia para criar um edge service" severity="secondary" />

## Passo 2: Crie um recurso

Para poder orquestrar serviços em seu dispositivo, você precisa configurar todos os recursos necessários para instalar, desinstalar e recarregar seus serviços.

Para criar um recurso, você precisa:

- Dar ao recurso um nome fácil de lembrar.
- Escolher o `content-type` do recurso. Exemplo: `text` e `shell-script`.
- Inserir o conteúdo do recurso.

<LinkButton link="/pt-br/documentacao/produtos/guias/deploy/provisionar-arquivos/" label="consulte o guia para criar um edge service resource" severity="secondary" />

---

## Passo 3: Gere um personal token

Para poder gerenciar edge nodes, você deve gerar um personal token.

Para fazer isso: 

- Dê um nome fácil de lembrar.
- Escreva uma descrição.
- Defina o tempo de expiração do token.

<LinkButton link="/pt-br/documentacao/produtos/guias/personal-tokens/" label="consulte o guia de criação de personal tokens" severity="secondary" />

---

## Passo 6: Vincule um edge service a um edge node

Agora, com o edge node criado e autorizado, você deve vinculá-lo a um edge service a ser implantado no edge node.

<LinkButton link="/pt-br/documentacao/produtos/guias/deploy/vincular-servico/" label="consulte o guia para vincular um edge service a um edge node" severity="secondary" />

Uma vez que essa relação estiver estabelecida, quando uma atualização for acionada nos edge services, ela será implantada em todos os nodes relacionados a este serviço em particular.
