Cada [conjunto de dados da GraphQL API](/pt-br/documentacao/devtools/graphql-api/recursos/#conjuntos-de-dados) possui campos específicos que podem ser utilizados ao consultar dados daquele conjunto de dados específico.

Veja cada campo disponível e suas descrições a seguir.

---

## accountingDetail (Accounting)

| Campo | Descrição |
| ----- | --------- |
| accounted | Quantidade total de uso de produtos por região contabilizada no detalhe da fatura do cliente. Exemplo: `1767504904` |
| clientId | Identificador único do cliente na Azion. Exemplo: `8437r` |
| metricSlug | Identificador da métrica sendo contabilizada. Consulte a lista completa de métricas de produtos no documento [Termos de Serviço](/pt-br/documentacao/contratos/tds/#3-metricas-e-metodologias-de-contabilizacao-de-uso-dos-produtos). Exemplo: `load_balancer_data_transferred` |
| periodFrom | Data de início dos dados contabilizados no formato ano-mês-dia. Exemplo: `2022-06-01` |
| periodTo | Data de término dos dados contabilizados no formato ano-mês-dia. Exemplo: `2022-06-30` |
| productSlug | Identificador do produto sendo contabilizado. Exemplo: `edge_functions` |
| regionName | Região em que a métrica foi contabilizada. Encontre mais informações na página de [Preços](/pt-br/documentacao/produtos/precos/). Pode ser: `United States`, `Canada`, `Europe`, `Brazil`, `Latam`, `All Other Regions` ou `No Region` |
