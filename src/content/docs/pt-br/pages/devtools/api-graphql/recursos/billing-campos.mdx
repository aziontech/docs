:::note
A Billing GraphQL API está disponível apenas para contas do tipo *Online Sales*.
:::

Cada [conjunto de dados da API GraphQL](/pt-br/documentacao/devtools/graphql-api/recursos/#conjuntos-de-dados) tem campos específicos que podem ser usados ao consultar dados desse conjunto de dados.

Veja cada campo disponível e suas descrições abaixo.

---

## balanceFinancialEntry (Entradas Financeiras)

| Campo | Descrição |
| ----- | --------- |
| clientId | Identificador único do cliente na Azion. Exemplo: `8437r` |
| entryType | Tipo de entrada financeira. Exemplo: `debit`, `trial_credit` |
| description | Descrição da entrada de saldo que será lançada na fatura do cliente. |
| amount | Valor registrado na entrada financeira de acordo com a moeda. Exemplo: `18750` |
| currency | Moeda utilizada na entrada financeira. Exemplo: `BRL` |

---

## paymentsClientDebt (Dívidas do Cliente)

| Campo | Descrição |
| ----- | --------- |
| clientId | Identificador único do cliente na Azion. Exemplo: `8437r` |
| created | Data em que a dívida do cliente foi criada/gerada no formato ano-mês-dia. Exemplo: `2023-07-31` |
| amount | Valor total contabilizado para a dívida do cliente de acordo com a moeda. Exemplo: `625` |
| currency | Moeda utilizada para a dívida do cliente. Exemplo: `BRL` |
| startDate | Data de início dos dados contabilizados no formato ano-mês-dia. Exemplo: `2022-06-01` |
| endDate | Data de término dos dados contabilizados no formato ano-mês-dia. Exemplo: `2022-06-30` |
| paymentDate | Data para o pagamento do cliente no formato ano-mês-dia. Exemplo: `2023-07-10` |
| cardBrand | Marca do cartão utilizado pelo cliente para o pagamento. Exemplo: `VISA` |
| cardLast4Digits | Últimos 4 (quatro) dígitos do cartão do cliente. Exemplo: `4456` |

---

## billDetail (Detalhes da Fatura)

| Campo | Descrição |
| ----- | --------- |
| clientId | Identificador único do cliente. Exemplo: `0001a` |
| periodFrom | Data de início dos dados contabilizados no formato ano-mês-dia. Exemplo: `2023-07-01` |
| periodTo | Data de término dos dados contabilizados no formato ano-mês-dia. Exemplo: `2023-07-31` |
| productSlug | Identificador do produto. Exemplo: `edge_functions` |
| metricSlug | Identificador da métrica. Exemplo: `invocations` |
| regionName | Região onde a métrica foi contabilizada. Exemplo: `Brazil` |
| accounted | Quantidade total de utilização contabilizada para o detalhe da fatura. Exemplo: `1767504904` |
| billId | Identificador único da fatura. Exemplo: `1000` |
| billDetailId | Identificador único do detalhe da fatura. Exemplo: `100000` |
| createdDate | Data de criação da fatura no formato ano-mês-dia. Exemplo: `2023-07-01` |
| temporaryBill | Indica se a fatura é referente ao mês atual (temporária). Exemplo: `false` |
| invoiceNumber | Identificador único da nota fiscal gerada para a fatura do cliente. Exemplo: `BRLTDA-2769z072023` |
