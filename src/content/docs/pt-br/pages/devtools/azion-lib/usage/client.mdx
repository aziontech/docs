O **Client** da Azion fornece uma interface unificada para interagir com todos os produtos e serviços. Você pode usar o `client` para acessar e gerenciar todas as funcionalidades, incluindo Storage, SQL, Purge e mais.

Ao instanciar um `client`, você pode definir configurações como `token` e `debug` explicitamente como parâmetros. Você pode então interagir com as funcionalidades da Azion diretamente através do `client`, de maneira simplificada e centralizada.

## Usando o Azion Client

Para usar a interface **client**, você precisa importar a função `createClient` da **Azion Lib**. Você pode então passar seu `token` como um parâmetro e usar a instância do `client` para acessar módulos e suas funcionalidades.

O exemplo abaixo mostra uma implementação em **JavaScript**:

```js
import { createClient } from 'azion';

// Instanciar o cliente
const client = createClient({ token: 'your-api-token', debug: true });

// Acessar o módulo SQL e criar um Banco de Dados
const { data: newDatabase, error } = await client.sql.createDatabase('my-new-database');
if (data) {
  console.log(`Database created with ID: ${newDatabase.id}`);
} else {
  console.error('Failed to create database', error);
}
```

Se você estiver usando TypeScript, pode importar os tipos apropriados como no exemplo abaixo:

```ts
import { createClient } from 'azion';
import type { AzionClient } from 'azion/client';
import type { AzionDatabaseResponse } from 'azion/sql';

// Instanciar o cliente
const client = createClient({ token: 'your-api-token', debug: true });

// Acessar o módulo SQL e criar um Banco de Dados
const { data: newDatabase, error }: AzionDatabaseResponse<AzionDatabase> = await client.sql.createDatabase('my-new-database');
if (data) {
  console.log(`Database created with ID: ${newDatabase.id}`);
} else {
  console.error('Failed to create database', error);
}
```

## Azion Client vs Funções de pacotes independentes

Alternativamente, se você preferir usar funções individuais diretamente de cada pacote, você precisa configurar tokens e configurações via variáveis de ambiente (por exemplo, usando um arquivo `.env`).

Cada módulo tem seu próprio `client` interno que gerencia as interações. O exemplo a seguir mostra o uso de um `client` para um módulo específico:

```js
import { createClient, StorageClient } from 'azion/storage';

// Criar um cliente para o módulo de Storage
const client: StorageClient = createClient({ token: 'your-api-token', debug: true });

const { data, error }: AzionStorageResponse<AzionBucket> = await client.createBucket({
  name: 'my-new-bucket',
  edge_access: 'public',
});

if (data) {
  console.log(`Bucket created with name: ${data.name}`);
} else {
  console.error('Failed to create bucket', error);
}
```

Também é possível usar funções específicas diretamente de seus pacotes, sem usar um `client`, como mostrado no exemplo abaixo.

:::note
Ao usar funções individuais sem um `client`, você precisa configurar tokens e configurações via variáveis de ambiente (por exemplo, usando um arquivo `.env`).
:::

```js
import { createDatabase } from 'azion/sql';

// Chamar a função createDatabase diretamente do seu pacote
const { data, error } = await createDatabase('my-new-database', { debug: true });
if (data) {
  console.log(`Database created with ID: ${data.id}`);
} else {
  console.error('Failed to create database', error);
}
```

Essa flexibilidade permite que você gerencie tudo de modo simplificado através do `client` ou chame funções específicas de cada pacote, com mais controle sobre as configurações do ambiente.
