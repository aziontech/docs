Veja o comando que permite o gerenciamento de [personal tokens](/pt-br/documentacao/produtos/gestao-de-contas/personal-tokens/) por meio da Azion CLI.

---

## create personal-token 

### Uso

```sh
azion create personal-token  --name "novoToken" --expiration "9m" 
```

### Flags opcionais

#### file 

A opção `--file` informa o caminho do arquivo JSON que contém todos os atributos do personal token que está sendo criado.

**Exemplo**

```sh
{
    "name": "Token de um dia",
    "expires_at": "9m"
}
```

#### help

A opção `--help` exibe mais informações sobre o subcomando `create`.

### Flags obrigatórias quando --file não é informado

#### description 

A opção `--description` informa a descrição do personal token.

#### expiration 

A opção `--expiration` informa a data de expiração do personal token.

**Exemplo** 

```sh 
azion create personal-token  --name "xxtoken" --expiration "9m" // meses
azion create personal-token  --name "xxtoken" --expiration "9d" // dias
azion create personal-token  --name "xxtoken" --expiration "1a" // ano
```

#### name

A opção `--name` informa o nome do personal token.

## list personal-token 

### Uso

```sh
azion list personal-token  [flags]
```

### Flags opcionais

#### details

A opção `--details` exibe todos os campos relevantes ao listar.

#### help

A opção `--help` exibe mais informações sobre o subcomando `list`.

--- 

## delete personal-token

### Uso

```sh
azion delete personal-token --id xxxxxx-xxxxx-xxxx-xxxxx-xxxx
```

### Flags obrigatórias 

#### id

A opção `--id` informa o identificador único do personal token que está sendo excluído.

#### Flags opcionais

#### help

A opção `--help` exibe mais informações sobre o subcomando `delete`.
