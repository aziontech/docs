Define arquivos estáticos de um deploy anterior dentro do mesmo bucket usando o Azion CLI.

:::note
Caso um ou mais campos obrigatórios não sejam informados através das flags específicas, uma mensagem interativa pedirá as informações faltantes.
:::

---

## Uso

```bash
azion rollback [flags]
```

## Exemplos

```bash
$ azion rollback --connector-id aaaa-bbbb-cccc-dddd
```

## Flags opcionais

#### connector-id

A opção `--connector-id` especifica o ID do conector do storage connector usado para arquivos estáticos.

#### config-dir

A opção `--config-dir` especifica o caminho relativo para onde seus arquivos personalizados `azion.json` e `args.json` estão armazenados. O padrão é "azion".

#### help

A opção `--help` exibe mais informações sobre o comando `rollback`.

## Opções globais

#### config

A opção `-c, --config` define a pasta de configuração da Azion apenas para o comando atual, sem alterar as configurações persistentes.

#### debug

A opção `-d, --debug` exibe logs no nível debug.

#### format

A opção `--format` altera o formato de saída passando o valor json para a flag.

#### log-level

A opção `-l, --log-level` define o nível de log: "debug", "info" ou "error". O padrão é "info".

#### no-color

A opção `--no-color` altera o formato de saída passando o valor json para a flag.

#### out

A opção `--out` exporta a saída para o `<caminho_arquivo/nome_arquivo.ext>` informado.

#### silent

A opção `-s, --silent` silencia completamente o log; usada principalmente para fins de automação.

#### timeout

A opção `--timeout` define quanto tempo em segundos a CLI aguardará antes de expirar a conexão HTTP. O padrão é 50 segundos.

#### token

A opção `-t, --token` salva um Personal Token localmente para autorizar comandos da CLI.

#### yes

A opção `-y, --yes` responde automaticamente "sim" para todas as interações de sim/não.
