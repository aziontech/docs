Azion CLI Warmup permite que você pré-carregue URLs no cache de edge por meio do crawling sistemático de uma URL, melhorando a performance das suas aplicações.

---

## Warmup

Realiza o cache warming por meio do crawling sistemático de uma URL e pré-carrega seu conteúdo no cache de edge.

### Uso

```bash
azion warmup --url "https://example.com"
```

```bash
azion warmup --url "https://example.com/products"
```

```bash
azion warmup --url "https://example.com/blog" --max-urls 500 --max-concurrent 5 --timeout 10000
```

### Flags obrigatórias

#### url

A opção `--url` especifica a URL base para iniciar o cache warming. Este é o ponto de entrada a partir do qual o comando fará o crawling e pré-carregará o conteúdo.

### Flags opcionais

#### help

A opção `--help` exibe mais informações sobre o comando warmup.

#### max-concurrent

A opção `--max-concurrent` define o número máximo de requisições concorrentes durante o processo de warming. O valor padrão é `2`.

#### max-urls

A opção `--max-urls` define o número máximo de URLs a serem processadas durante o cache warming. O valor padrão é `1500`.

#### timeout

A opção `--timeout` especifica o tempo limite em milissegundos para cada requisição. O valor padrão é `8000`.

## Flags globais

As seguintes flags globais podem ser usadas com o comando warmup:

| Flag | Descrição |
|------|-----------|
| `-c, --config` | Define a pasta de configuração do Azion para o comando atual, sem alterar as configurações persistentes. |
| `-d, --debug` | Exibe logs no nível de debug. |
| `--format` | Altera o formato de saída passando o valor json para a flag. |
| `-l, --log-level` | Define o nível de log: "debug", "info" ou "error". O padrão é "info". |
| `--no-color` | Altera o formato de saída passando o valor json para a flag. |
| `--out` | Exporta a saída para o `<caminho_arquivo/nome_arquivo.ext>` informado. |
| `-s, --silent` | Silencia completamente os logs; usado principalmente para automação. |
| `-t, --token` | Salva um Personal Token localmente para autorizar comandos CLI. |
| `-y, --yes` | Responde automaticamente todas as interações de sim/não com sim. |
