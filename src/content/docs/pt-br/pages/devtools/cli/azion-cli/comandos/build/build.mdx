Faça a build de sua application localmente e inicie a preparação para a implatação no edge.

:::note
No caso de um ou mais campos obrigatórios não serem informados através das flags específicas, uma mensagem interativa será exibida, solicitando essas informações.
:::

---

## Uso

```bash
  azion build
```

## Flags opcionais

#### help

A opção `--help` exibe mais informações sobre o comando `build`.

#### config-dir

A opção `--config-dir` especifica o caminho relativo onde seus arquivos personalizados `azion.json` e `args.json` estão armazenados. O valor padrão é `azion`.

#### entry

A opção `--entry` especifica o ponto de entrada do código para a build. O valor padrão é `./main.js`.

#### preset

A opção `--preset` pode ser especificada se for necessário alterar o preset durante o processo de criação, mas não é obrigatória. Os valores esperados são:

- Html
- Javascript
- Typescript
- Angular
- Astro
- Hexo
- Next
- React
- Vue
- static

#### skip-framework-build

A opção `--skip-framework-build` indica se deve ignorar a fase de build do framework antes de executar `azion build`. Isso é útil quando você já construiu seu framework separadamente e deseja pular essa etapa.

#### use-node-polyfills

A opção `--use-node-polyfills` especifica se deve usar polyfills de Node no processo de build. Isso ajuda a garantir compatibilidade ao usar APIs específicas do Node.js em ambientes de edge.

#### use-own-worker

A opção `--use-own-worker` indica que o código construído insere sua própria expressão de worker, como `addEventListener("fetch")` ou similar, sem a necessidade de injetar um provider. Isso é útil quando seu código já gerencia a configuração do worker.
