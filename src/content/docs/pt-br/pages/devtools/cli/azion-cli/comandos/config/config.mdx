Gerencie recursos de configuração do Azion a partir de arquivos manifest. O comando `config` fornece subcomandos para inicializar, aplicar e excluir configurações.

---

:::note
**Versões da API: Contas criadas após 2026-01-01 usam automaticamente a API v4, enquanto contas mais antigas podem usar a API v3. A CLI detecta automaticamente a versão correta.

Para verificar qual conta está sendo utilizada, execute `azion whoami`.
:::

---

## Uso

```sh
azion config [flags]
azion config [subcommand] [flags]
```

## Subcomandos disponíveis

| Subcomando | Descrição |
|------------|-----------|
| `init` | Inicializa o arquivo de configuração do Azion |
| `apply` | Aplica a configuração do arquivo de configuração para a Plataforma Azion |
| `delete` | Exclui todos os recursos do azion.json |

---

## config init

Cria um arquivo `azion.json` no diretório atual para armazenar o estado da sua aplicação.

### Uso

```sh
azion config init [flags]
```

### Exemplos

```sh
azion config init
azion config init --config-dir ./meu-projeto
```

### Flags opcionais

#### config-dir

A opção `--config-dir` especifica o caminho para o diretório onde o `azion.json` será criado. O padrão é o diretório atual.

```sh
azion config init --config-dir ./meu-projeto
```

#### force

A opção `--force` sobrescreve um arquivo de configuração existente sem solicitar confirmação.

```sh
azion config init --force
```

#### help

A opção `--help` exibe mais informações sobre o subcomando `init`.

---

## config apply

Aplica os recursos de configuração definidos no arquivo `azion.config` para a Plataforma Azion.

### Uso

```sh
azion config apply [flags]
```

### Exemplos

```sh
azion config apply
azion config apply --config-dir ./meu-projeto
```

### Flags opcionais

#### config-dir

A opção `--config-dir` especifica o caminho para o diretório de configuração contendo `azion.json` e `azion.config`. O padrão é o diretório atual.

```sh
azion config apply --config-dir ./meu-projeto
```

#### help

A opção `--help` exibe mais informações sobre o subcomando `apply`.

---

## config delete

Exclui todos os recursos encontrados no arquivo de configuração `azion.json` e o reinicia para um estado limpo.

### Uso

```sh
azion config delete [flags]
```

### Exemplos

```sh
azion config delete
azion config delete --force
azion config delete --config-dir ./meu-projeto
```

### Flags opcionais

#### config-dir

A opção `--config-dir` especifica o caminho relativo onde seu arquivo `azion.json` está armazenado. O padrão é o diretório atual.

```sh
azion config delete --config-dir ./meu-projeto
```

#### force

A opção `--force` força a exclusão sem prompt de confirmação.

```sh
azion config delete --force
```

#### help

A opção `--help` exibe mais informações sobre o subcomando `delete`.

## Flags globais

| Flag | Descrição |
|------|-----------|
| `-c, --config string` | Define a pasta de configuração do Azion apenas para o comando atual, sem alterar as configurações persistentes. |
| `-d, --debug` | Exibe logs no nível de debug. |
| `--format string` | Altera o formato de saída passando o valor json para a flag. |
| `-l, --log-level string` | Define o nível de logging: "debug", "info" ou "error". O padrão é "info". |
| `--no-color` | Altera o formato de saída passando o valor json para a flag. |
| `--out string` | Exporta a saída para o `<caminho_arquivo/nome_arquivo.ext>` fornecido. |
| `-s, --silent` | Silencia completamente os logs; usado principalmente para automação. |
| `--timeout int` | Define quanto tempo em segundos a CLI aguardará antes de encerrar a conexão HTTP. O padrão é 50. |
| `-t, --token string` | Salva um Personal Token localmente para autorizar comandos da CLI. |
| `-y, --yes` | Responde automaticamente "sim" para todas as interações de sim/não. |
