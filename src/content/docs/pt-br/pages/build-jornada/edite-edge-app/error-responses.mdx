:::caution[importante]
A Azion possui duas interfaces de usuário: Real-Time Manager e Console, que está em estágio de *Preview*. Atualmente, o Console está disponível apenas para planos de Developer e novas contas. Siga os passos de acordo com a interface de usuário que estiver utilizando.
:::

Configure sua edge application para lidar com status de erros da origem usando [Error Responses](/pt-br/documentacao/produtos/build/edge-application/error-responses/). A Azion permite que você determine um TTL de cache para páginas de erro, um caminho personalizado, ou entregue outro código HTTP na resposta sem precisar alterar o código-fonte de sua aplicação.

Com este guia, você modificará a resposta de erro padrão de sua aplicação, determinando um TTL de cache para cada erro recebido da origem.

<Fragment slot="panel.console">
1. Acesse o [Azion Console](/pt-br/documentacao/produtos/guias/como-acessar-o-azion-console/) > **Edge Application**.
2. Clique na edge application que você deseja configurar.
3. Selecione a aba **Error Responses**.

Na seção **Error Responses**, o primeiro campo **Status Code** não pode ser modificado. Este campo representa quaisquer códigos de erro recebidos da origem. O campo **Default Response TTL** abaixo dele permite que você personalize o TTL de cache para todos os códigos recebidos.

4. Em **Default Response TTL**, defina o *time-to-live* (TTL) do cache, em segundos, para todas as páginas de erro.
5. Clique no botão **+ Error Response**.
6. No novo campo **Status Code**, selecione o código de status que você deseja personalizar. Por exemplo: `404`.
7. Adicone um TTL de cache no campo **Custom Response TTL**.

Você também pode determinar um caminho específico da sua origem que será entregue ao usuário e um código de status personalizado na resposta. Essas modificações são opcionais.

8. No campo **Page Path**, adicione um *path* para uma página de erro criada na fonte. Por exemplo: `/error/404`.
9. No campo **Response Status Code*, selecione o status HTTP que será recebido pelo usuário.
10. Na seção **Set Origin**, selecione a origem das URIs da página de erro.
11. Clique no botão **Save**.

Quando erros forem recebidos da origem, sua edge application irá armazenar em cache e exibir a página de erro com base na nova configuração.
</Fragment>

<Fragment slot="panel.rtm">
1. Acesse o [Real-Time Manager (RTM)](https://manager.azion.com/).
2. No canto superior esquerdo da página, abra o **Products menu**, representado por três linhas horizontais, e selecione **Edge Application**.
3. Clique na edge application que você deseja configurar.
4. Selecione a aba **Error Responses**.

Na seção **Error Responses**, o primeiro campo **Status Code** não pode ser modificado. Este campo representa quaisquer códigos de erro recebidos da origem. O campo **Error Caching TTL (seconds)** abaixo dele permite que você personalize o TTL de cache para todos os códigos recebidos.

5. Em **Error Caching TTL (seconds)**, defina o TTL do cache, em segundos, para todas as páginas de erro.
6. Clique no botão **+**.
7. No novo campo **Status Code**, selecione o código de status que você deseja personalizar. Por exemplo: `404`.
8. Adicone um TTL de cache no campo **Error Caching TTL (seconds)**.

Você também pode determinar um caminho específico da sua origem que será entregue ao usuário e um código de status personalizado na resposta. Essas modificações são opcionais.

9. No campo **URI**, adicione um *path* para uma página de erro criada na fonte. Por exemplo: `/error/404`.
10. No campo **Custom Status Code**, selecione o status HTTP que será recebido pelo usuário.
11. Na seção **Origin**, selecione a origem das URIs da página de erro.
12. Clique no botão **Save**.

Quando erros forem recebidos da origem, sua edge application irá armazenar em cache e exibir a página de erro com base na nova configuração.
</Fragment>
