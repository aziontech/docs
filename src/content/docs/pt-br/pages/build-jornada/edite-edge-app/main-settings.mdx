Quando você [cria uma edge application com a Azion](/pt-br/documentacao/produtos/comecar-com-um-template/), você tem uma série de configurações que pode modificar. Neste guia, você aprenderá a modificar os protocolos, portas de entrega e o tipo de criptografia que sua aplicação usa para se comunicar com seus usuários finais.

Nesta seção, você modificará algumas configurações do sua aplicação. Você pode ver se as alterações ocorreram [acessando seu domínio Azion](/pt-br/documentacao/produtos/guias/configurar-dominio/).

:::note
Ainda não tem uma edge application? [Comece com um template](/pt-br/documentacao/produtos/comecar-com-um-template/).
:::

<Fragment slot="panel.console">
    1. Acesse o [Azion Console](/pt-br/documentacao/produtos/guias/como-acessar-o-azion-console/) > **Edge Application**.
    2. Clique na edge application que você deseja configurar. A partir desta tela, você pode alterar o nome da aplicação, as configurações de entrega e quais add-ons ela usa.

    Os próximos passos irão guiá-lo na edição das configurações de uma edge application boilerplate. Sinta-se à vontade para alterar essas configurações para atender às suas necessidades.

   **Add-ons**

    Os [Add-ons de Edge Application](/pt-br/documentacao/produtos/build/edge-application/#modules) podem desbloquear funcionalidades e recursos adicionais para suas aplicações. Veja a [página de preços](https://www.azion.com/pt-br/precos/) para mais detalhes.

    Por enquanto, você ativará dois add-ons. Os add-ons **Application Accelerator** e **Edge Functions** serão usados quando você [ajustar suas configurações de cache](/pt-br/documentacao/produtos/guias/build/ajustar-cache-settings/), [instanciar uma edge function](/pt-br/documentacao/produtos/guias/build/instanciar-edge-functions/) e [Trabalhar com Rules Engine](/pt-br/documentacao/produtos/guias/build/trabalhar-com-rules-engine/). 

    Em seguida, para habilitar os add-ons para o sua aplicação:

    3. Vá para a aba **Main Settings**.
    4. Em **Add-ons**, selecione e ative **Application Accelerator** e **Edge Functions**.
    5. Clique no botão **Save**.
</Fragment>

<Fragment slot="panel.api">
    1. Execute a seguinte requisição `GET` no seu terminal, substituindo `[TOKEN VALUE]` pelo seu [personal token](/pt-br/documentacao/produtos/guias/personal-tokens/) para recuperar seu `<edge_application_id>`:

    ```bash
    curl --location 'https://api.azionapi.net/v4/edge_application/applications' \
    --header 'Accept: application/json;' \
    --header 'Authorization: Token [TOKEN VALUE]'
    ```

    2. Você receberá uma resposta com todas sua edge applications existentes. Copie o valor do `<edge_application_id>` que você deseja configurar.
    3. Execute uma requisição `PATCH` para modificar a aplicação da seguinte forma:

    ```bash
    curl --location --request PATCH 'https://api.azionapi.net/v4/edge_application/applications/<application_id>' \
    --header 'Accept: application/json;' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Token [TOKEN VALUE]' \
    --data '{
       
        "edge_functions": true,
        "application_acceleration": true
    }'
    ```


    | Chave | Descrição |
    | --- | --- |
    | `edge_functions` | Habilita o add-on **Edge Functions** |
    | `application_acceleration` | Habilita o add-on **Application Accelerator** |

    4. Você receberá uma resposta semelhante a esta:

    ```json
    {
        "state": "executed",
        "data": {
            "id": <edge_application_id>,
            "name": "string",
            "last_editor": "string",
            "last_modified": "2019-08-24T14:15:22Z",
            "add-ons": {
                "edge_cache_enabled": true,
                "edge_functions_enabled": true,
                "application_accelerator_enabled": true,
                "image_processor_enabled": false,
                "tiered_cache_enabled": false
            },
        "active": true,
        "debug": false,
        "product_version": "string"
        }
    }
    ```

    5. Aguarde alguns minutos para que as alterações se propaguem.
    6. Teste suas alterações acessando sua aplicação. Você pode acessar sua aplicação através de um navegador usando o endereço `http://xxxxxxxxxx.map.azionedge.net:8008` e inspecionar a página ou usar o seguinte comando cURL para uma resposta de status `2xx` ou `3xx`:

    ```bash
    curl -I http://xxxxxxxxxx.map.azionedge.net:8008
    ```

    7. Verifique o conjunto de cifras suportado inspecionando a página ou [executando o comando DIG](/pt-br/documentacao/produtos/guias/executar-o-comando-dig/).
