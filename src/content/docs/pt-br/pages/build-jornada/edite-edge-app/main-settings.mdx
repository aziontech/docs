Quando você [cria uma application com a Azion](/pt-br/documentacao/produtos/comecar-com-um-template/), você tem uma série de configurações que pode modificar. Neste guia, você aprenderá a modificar os protocolos, portas de entrega e o tipo de criptografia que sua aplicação usa para se comunicar com seus usuários finais.

Nesta seção, você modificará algumas configurações do sua aplicação. Você pode ver se as alterações ocorreram [acessando seu domínio Azion](/pt-br/documentacao/produtos/guias/configurar-dominio/).

:::note
Ainda não tem uma application? [Comece com um template](/pt-br/documentacao/produtos/comecar-com-um-template/).
:::

<Fragment slot="panel.console">
    1. Acesse o [Azion Console](/pt-br/documentacao/produtos/guias/como-acessar-o-azion-console/) > **Applications**.
    2. Clique na application que você deseja configurar. A partir desta tela, você pode alterar o nome da aplicação, as configurações de entrega e quais módulos ela usa.

    Os próximos passos irão guiá-lo na edição das configurações de uma application boilerplate. Sinta-se à vontade para alterar essas configurações para atender às suas necessidades.

    **Delivery protocols**

    Modifique sua aplicação para suportar os [protocolos de entrega](/pt-br/documentacao/produtos/build/edge-application/main-settings/#protocolos-de-entrega) disponíveis.

    3. Vá para a aba **Main Settings**.
    4. Em **Protocol Usage**, selecione **HTTP support**.
    5. Clique no botão **Save**.

    Após algum tempo de propagação, você poderá acessar sua application usando o protocolo HTTP. Para testar o funcionamento do novo protocolo, acesse o endereço `http://xxxxxxxxxx.map.azionedge.net`. Você deverá acessar a aplicação sob um protocolo mais seguro.

    Você também pode fazer a seguinte requisição cURL no seu terminal com o protocolo HTTP:

    ```bash
    curl -I http://xxxxxxxxxx.map.azionedge.net/
    ```

    Uma resposta `2xx` ou `3xx` significa que sua aplicação está rodando no protocolo HTTP.

    <MainSettingsSAN />

    **Delivery ports**

    Escolha entre as [portas de entrega](/pt-br/documentacao/produtos/build/edge-application/main-settings/#portas) suportadas para cada protocolo selecionado.

    6. Vá para a aba **Main Settings**.
    7. Em **HTTP Ports**, além da porta `80`, selecione a porta `8008`.
    8. Clique no botão **Save**.

    Aguarde alguns minutos para que as alterações se propaguem para o edge e tente acessar sua aplicação usando uma das novas portas. Por exemplo, acesse `http://xxxxxxxxxx.map.azionedge.net:8008` e verifique se sua aplicação está rodando na porta personalizada.

    Você também pode fazer a seguinte requisição cURL do seu terminal:

    ```bash
    curl -I http://xxxxxxxxxx.map.azionedge.net:8008
    ```

    Uma resposta bem-sucedida significa que sua aplicação também está rodando na porta `8008`.

   **Módulos**

    Os [Módulos de Applications](/pt-br/documentacao/produtos/build/edge-application/#modules) podem desbloquear funcionalidades e recursos adicionais para suas aplicações. Veja a [página de preços](https://www.azion.com/pt-br/precos/) para mais detalhes.

    Por enquanto, você ativará dois módulos. Os módulos **Application Accelerator** e **Functions** serão usados quando você [ajustar suas configurações de cache](/pt-br/documentacao/produtos/guias/build/ajustar-cache-settings/), [instanciar uma function](/pt-br/documentacao/produtos/guias/build/instanciar-edge-functions/) e [Trabalhar com Rules Engine](/pt-br/documentacao/produtos/guias/build/trabalhar-com-rules-engine/). 

    Se você é um [account owner](/pt-br/documentacao/produtos/guias/users-management/), comece ativando os seguintes módulos para sua conta:

    9. [Acesse o Azion Console](https://console.azion.com).
    10. No canto superior direito, clique no **Account menu**.
    11. Selecione **Billing & Subscriptions**.
    12. Navegue até a aba **Subscriptions**.
    13. Ative **Application Accelerator** e **Functions**.
    14. Clique no botão **Save**.

    <MainSettingsUpdatePayments />

    Em seguida, para habilitar os módulos para o sua aplicação:

    15. No canto superior esquerdo da página, abra o **Products menu**, representado por três linhas horizontais, e selecione **Applications**.
    16. Clique na application que você deseja configurar.
    17. Ative os módulos **Application Accelerator** e **Functions**.
    18. Clique no botão **Save**.
</Fragment>

<Fragment slot="panel.api">
    1. Execute a seguinte requisição `GET` no seu terminal, substituindo `[TOKEN VALUE]` pelo seu [personal token](/pt-br/documentacao/produtos/guias/personal-tokens/) para recuperar seu `<application_id>`:

    ```bash
    curl --location 'https://api.azionapi.net/edge_applications \
    --header 'Accept: application/json; version=3' \
    --header 'Authorization: Token [TOKEN VALUE]'
    ```

    2. Você receberá uma resposta com todas sua applications existentes. Copie o valor do `<application_id>` que você deseja configurar.
    3. Execute uma requisição `PATCH` para modificar a aplicação da seguinte forma:

    ```bash
    curl --location --request PATCH 'https://api.azionapi.net/edge_applications/<application_id>' \
    --header 'Accept: application/json; version=3' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Token [TOKEN VALUE]' \
    --data '{
        "delivery_protocol": "http",
        "http_port": [80, 8008],
        "edge_functions": true,
        "application_acceleration": true
    }'
    ```


    | Chave | Descrição |
    | --- | --- |
    | `delivery_protocol` | Quando definido como `http`, habilita o suporte ao protocolo HTTP |
    | `http_port` | Define portas HTTP para a aplicação. Neste caso, o array `[80, 8008]` define que ambas as portas 80 e 8008 podem ser usadas por solicitações HTTP |
    | `edge_functions` | Habilita o módulo **Functions** |
    | `application_acceleration` | Habilita o módulo **Application Accelerator** |

    4. Você receberá uma resposta semelhante a esta:

    ```json
    {
        "results": {
            "id": <application_id>,
            "name": "example.org",
            "delivery_protocol": "http",
            "http_port": [
                80,
                8008
            ],
            "https_port": [],
            "minimum_tls_version": "",
            "active": true,
            "debug_rules": false,
            "http3": false,
            "websocket": false,
            "supported_ciphers": "all",
            "application_acceleration": true,
            "caching": true,
            "device_detection": false,
            "edge_firewall": false,
            "edge_functions": true,
            "image_optimization": false,
            "l2_caching": false,
            "load_balancer": false,
            "raw_logs": false,
            "web_application_firewall": false
        }
    }
    ```

    5. Aguarde alguns minutos para que as alterações se propaguem.
    6. Teste suas alterações acessando sua aplicação usando o protocolo HTTP e as portas personalizadas configuradas acima. Você pode acessar sua aplicação através de um navegador usando o endereço `http://xxxxxxxxxx.map.azionedge.net:8008` e inspecionar a página ou usar o seguinte comando cURL para uma resposta de status `2xx` ou `3xx`:

    ```bash
    curl -I http://xxxxxxxxxx.map.azionedge.net:8008
    ```

    7. Verifique o conjunto de cifras suportado inspecionando a página ou [executando o comando DIG](/pt-br/documentacao/produtos/guias/executar-o-comando-dig/).
