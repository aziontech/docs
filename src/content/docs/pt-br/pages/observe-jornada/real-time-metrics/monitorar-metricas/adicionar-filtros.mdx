O **Real-Time Metrics** mostra uma visão padrão dos dados relacionados a todas suas edge applications. Ao usar os filtros, você pode ajustar os gráficos para corresponder às suas requisições e visualizar dados mais específicos que se encaixam na sua análise.

Ao adicionar um filtro, você pode definir o **campo**, como "host", "status" ou "schema", por exemplo. Os campos são usados com os **operadores** disponíveis, como "Like", "Range" ou "Ne".

Você pode encontrar mais detalhes sobre cada campo na [documentação de Campos](/pt-br/documentacao/produtos/graphql-api/recursos/campos-metrics/) e sobre operadores na [documentação de Operadores](/pt-br/documentacao/produtos/graphql-api-queries/#operadores).

Você também pode definir o **campo de valor** que será usado. Dependendo da variável escolhida, ela pode aceitar valores dos tipos String, Int ou Float.

### Exemplo prático

Você quer filtrar seus dados pela quantidade de dados economizados, mas, na requisição, você quer definir um valor específico como início para os dados que serão retornados.

Nesta situação, você deve usar a variável **savedDataGte**. No campo de valor, você deve adicionar o valor numérico que deseja usar como ponto de corte, como "8300". Assim, sua resposta conterá apenas quantidades de dados economizados que começam e são maiores do que o valor específico que você definiu.

> **Como posso editar ou excluir um filtro?**
>
> Depois de adicionar filtros, cada um é exibido na seção **FILTERS**.
>
> Se você quiser editar um filtro existente, clique no texto do filtro específico. Assim que o popover **Edit Filter** abrir, você poderá alterar as configurações do filtro conforme desejar.
>
> Se você quiser excluir um filtro existente, clique no ícone **x** ao lado do filtro específico. Repita a ação para cada filtro que você deseja excluir.
