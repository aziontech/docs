Ao criar ou editar um streaming, existem algumas configurações principais específicas que você pode definir e modificar.

Para obter informações mais detalhadas sobre o produto e cada configuração, consulte a [documentação de referência](/pt-br/documentacao/produtos/data-streaming/).

---

## Via RTM

Nesta seção, você modificará as configurações principais de sua transmissão: nome, fonte de dados, modelo, domínios e destino via [Real-Time Manager (RTM)](https://manager.azion.com/).

1. [Acesse o RTM](/pt-br/documentacao/produtos/guias/como-acessar-o-rtm/) > **Data Streaming**.
2. Na lista, selecione o streaming que deseja editar ou clique no botão **Add Streaming**.
3. Dê ao streaming um nome único e fácil de lembrar.
4. No menu suspenso **Data Source**, selecione o que deseja usar: **Activity History**, **Edge Applications**, **Edge Functions** ou **WAF Events**.
    - Para usar Edge Functions e WAF Events, você deve [ter os produtos ativados](/pt-br/documentacao/produtos/data-streaming/#selecionar-data-source).
5. No campo **Template**, selecione o que desejar usar: um template pré-configurado de acordo com sua escolha de data source ou um **Custom Template**.
    - Caso escolha **Custom Template**, na caixa de código **Data Set**, adicione as variáveis que você deseja usar nas análises de seus logs. Veja mais sobre [Criar um template personalizado no Data Streaming](/pt-br/documentacao/produtos/guias/data-streaming-template-personalizado/).
6. Em **Options**, selecione **Filter Domains** ou **All Domains**.
    - Saiba mais sobre cada opção em [Como associar domínios no Data Streaming](/pt-br/documentacao/produtos/guias/data-streaming-associar-dominios/).
7. Na seção **Destination**, selecione um **Endpoint Type** na lista suspensa: **Standard HTTP/HTTPS POST**, **Apache Kafka**, **Simples Storage Service (S3)**, **Google BigQuery**, **Elasticsearch**, **Splunk**, **AWS Kinesis Data Firehose**, **Datadog**, **IBM QRadar**, **Azure Monitor** ou **Azure Blob Storage**.
    - Os campos para preenchimento são diferentes dependendo do tipo de endpoint que você escolher. Descubra mais informações sobre cada campo na página de [configuração de endpoint](/pt-br/documentacao/produtos/data-streaming/#endpoints).
8. Clique no botão **Save**.

## Via API

Nesta seção, você estará modificando um streaming existente com um conector Elasticsearch via API. Para criar um novo, use o [método POST do Data Streaming](https://api.azion.com/#b9aac0c7-173a-462b-87f6-a18d6558d3e5).

:::tip
Consulte a tabela de [Data Streaming Properties](https://api.azion.com/#b9aac0c7-173a-462b-87f6-a18d6558d3e5) para ver todas as propriedades disponíveis e seus valores aceitos.
:::

1. Execute a seguinte requisição `GET` em seu terminal, substituindo `[TOKEN VALUE]` pelo seu [personal token](/pt-br/documentacao/produtos/guias/personal-tokens/) para recuperar seu `<data_streaming_id>`:

```bash
curl --location 'https://api.azionapi.net/data_streaming/streamings' \
--header 'Accept: application/json; version=3' \
--header 'Authorization: Token [TOKEN VALUE]'
```

2. Você receberá uma resposta com todos os seus streamings existentes. Copie o valor do `<id>` que deseja configurar.
3. Execute uma requisição `PATCH` para modificar o streaming da seguinte forma:

```bash
curl --location --request PATCH 'https://api.azionapi.net/data_streaming/streamings/<data_streaming_id>' \
--header 'Accept: application/json; version=3' \
--header 'Authorization: Token [TOKEN VALUE]' \
--header 'Content-Type: application/json' \
--data '{
	"name": "My Elasticsearch connector",
    "template_id": 2,
    "domain_ids": [1656613172],
    "data_source": "http",
    "endpoint": {
        "endpoint_type": "elasticsearch",
        "url": "http://elasticsearch-domain.com",
        "api_key": "VuaCfGcBCdbkQm-e5aOx:ui2lp2axTNmsyakw9tvNnw"
    },
    "all_domains": false
}'
```

| Chave | Descrição |
| --- | --- |
| `name` | Nome do streaming |
| `template_id` | Identificador do template sendo usado |
| `domain_ids` | Valor de array dos identificadores (integers) dos domínios que você deseja associar ao streaming |
| `data_source` | Data source do qual você deseja transmitir seus dados |
| `endpoint` | Endpoint para o qual você deseja transmitir seus dados |
| `all_domains` | Usado para associar todos os domínios atuais e futuros em sua conta ao streaming |

4. Você receberá uma resposta semelhante a esta:

```json
{
  "results": {
    "id": 1560,
    "name": "My Elasticsearch connector",
    "template_id": 2,
    "data_source": "http",
    "active": true,
    "endpoint": {
      "endpoint_type": "elasticsearch",
      "url": "http://elasticsearch-domain.com",
      "api_key": "VuaCfGcBCdbkQm-e5aOx:ui2lp2axTNmsyakw9tvNnw"
    },
    "all_domains": false
  },
  "schema_version": 3
}
```

Aguarde alguns minutos para que as alterações se propaguem e sua transmissão será atualizada.

:::tip
Consulte a [documentação da API da Azion](https://api.azion.com/) e a [especificação do OpenAPI](https://github.com/aziontech/azionapi-openapi/) para saber mais sobre o que a API da Azion pode oferecer.
:::
