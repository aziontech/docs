Os dados transmitidos pelo **Data Stream** para seus conectores são decididos de acordo com as variáveis que você escolhe usar. Cada variável fornece uma informação ou conjunto de informações específicos e está associada a um data source específico.

Você pode encontrar todas as variáveis disponíveis, suas descrições e exemplos na referência de [data sources](/pt-br/documentacao/produtos/observe/data-stream/#data-sources).

Você tem a opção de usar um [template pré-definido](#use-um-template-pre-definido), com uma opção para cada data source disponível, ou usar um [template personalizado](#use-um-template-personalizado), para que você possa decidir quais variáveis serão transmitidas.

---

## Use um template pré-definido

Se você estiver configurando um conector para receber o máximo de informações possível, por exemplo, de uma edge application, usar um template pré-definido pode ser a melhor escolha. Nesse caso, você usará o template **Edge Applications + WAF Event Collector** via Azion Console ou o template **http** via API.

## Use um template personalizado

Se você estiver configurando um conector com um propósito específico, como, por exemplo, analisar logs relacionados ao status de sua edge application, você pode configurar:

- Um template personalizável via Azion Console, editando o **Data Set** selecionado.
- Um **Custom Template** via Real-Time Manager.
- um **template_model** via API.

Criar um template personalizável permite que você escolha, da [lista de variáveis disponíveis](/pt-br/documentacao/produtos/observe/data-stream/#data-sources), aquelas que deseja usar.

Neste caso, você poderia usar, por exemplo, as variáveis `$status`, `$proxy_status` e `$upstream_status`. Você também poderá editar o stream e adicionar novas variáveis se perceber que elas não estão fornecendo todas as informações necessárias.
