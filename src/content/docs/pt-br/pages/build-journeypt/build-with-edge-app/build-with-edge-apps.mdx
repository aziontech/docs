Azion [Edge Application](/pt-br/documentacao/produtos/edge-application/) permite criar novas aplicações, bem como ajustar várias configurações de entrega e definir as variáveis de suas aplicações. Você também pode automatizar comportamentos no nível de requisição-resposta.

Azion fornece uma variedade de ferramentas para criar e configurar suas edges applications:

- [Real-Time Manager (RTM)](https://manager.azion.com): interface usada para configurar suas aplicações na Azion e monitorar dados.
- [Azion CLI](/pt-br/documentacao/produtos/cli/): permite que você interaja com o sistema Azion diretamente da interface de linha de comando.
- [Azion API](https://api.azion.com): permite que você integre e automatize processos de implantação, escalonamento e monitoramento de aplicações.
- [Terraform](/pt-br/documentacao/produtos/terraform-provider/): integra sua infraestrutura com Azion usando Terraform.
- [Azion SDK](/pt-br/documentacao/devtools/sdk/go/): melhora o desenvolvimento de sua aplicação Go.
- [Edge Runtime](/pt-br/documentacao/produtos/edge-application/edge-functions/runtime/overview/): fornece APIs para manipulação de rede e dados.

## Passo 1: Configurar uma aplicação

Depois de [criar sua edge application](/documentacao/produtos/comecar-com-um-template/), você pode alterar os protocolos, portas de entrega e o tipo de criptografia que ela usa para se comunicar com seus usuários finais.

Para protocolos, você pode escolher se deseja entregar sua aplicação via **HTTP ou HTTPS**. HTTP é amplamente usado em casos onde os dados transferidos não precisam de segurança robusta, enquanto HTTPS fornece um nível extra de segurança através de criptografia. Uma **porta de entrega** personalizada pode otimizar o tráfego de rede, ajudando você a gerenciar melhor o fluxo de dados, o que aumenta a velocidade e eficiência de sua rede, bem como melhora a segurança em suas redes tornando o ponto de entrada menos previsível.

Quanto à criptografia, a implementação do [Transport Layer Security (TLS)](/documentacao/produtos/guias/criar-certificado-digital/) para aplicações HTTPS garante que as transferências de dados entre dispositivos de usuários e seus servidores sejam privadas.

Comece a [personalizar suas configurações principais](/pt-br/documentacao/produtos/guias/build/configurar-main-settings/) e [vincule aplicação a um domínio](/pt-br/documentacao/produtos/guias/configurar-um-dominio/).

## Passo 2: Definir variáveis

Após definir as configurações inciais, você pode modificar o comportamento de sua aplicação definindo variáveis e parâmetros que podem ser personalizados para cada aplicação para atender aos requisitos de sua aplicação. Você pode definir as seguintes instâncias:

- **Configuração de cache**: ajuste as [configurações de cache no edge](/pt-br/documentacao/produtos/guias/build/ajustar-cache-settings/) para reduzir a carga nos servidores de origem e acelerar a entrega. Defina valores como expiração de cache de browser e edge, file slicing, cache keys e entrega adaptada com base em dispositivos.
- **Configuração de origem**: ajuste suas [configurações de origem](/pt-br/documentacao/produtos/guias/build/work-with-origins/) para atender às necessidades de sua aplicação e defina os valores do cabeçalho `Host` e endereço de origem.
- **Grupo de dispositivos**: agrupe usuários finais [com base em seus dispositivos](/pt-br/documentacao/produtos/guias/build/create-device-groups/), sistemas operacionais ou navegadores com informações fornecidas pelo cabeçalho `User-Agent`.
- **Edge functions**: instancie [edge functions](/pt-br/documentacao/produtos/guias/build/instantiate-edge-functions/) para minimizar o uso de recursos processando no edge.

Algumas variáveis podem já ter sido criadas por padrão quando você construiu sua aplicação pela primeira vez. Você pode editar suas configurações padrão ou criar novas, se necessário.

## Passo 3: Ativar configurações

Para usar suas variáveis, você precisa [configurar regras](/pt-br/documentacao/produtos/guias/build/work-with-rules-engine/) que acionam comportamentos específicos em sua aplicação.

Tanto nas etapas de processamento de requisição quanto de resposta, você pode personalizar um conjunto de regras para lidar com requisições de acordo com os requisitos de sua aplicação. Isso permite que você automatize respostas para interações de usuários, gerencie o fluxo de dados e execute tarefas com base em requisições recebidas ou respostas enviadas, todas governadas por lógica condicional.
