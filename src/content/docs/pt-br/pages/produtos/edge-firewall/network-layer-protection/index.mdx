**Network Layer Protection** é um módulo de Edge Firewall que permite a criação de listas com base no IP/CIDR da rede, localização do usuário ou Autonomous System Number (ASN), ou usar listas automáticas mantidas e atualizadas pela Azion, como uma lista de endereços na rede Tor.

Com Network Layer Protection, você pode monitorar e bloquear comportamentos suspeitos ou aplicar restrições, como limites de acesso.

Network Layer Protection oferece um perímetro de segurança programável, onde você pode configurar a proteção de rede, tanto para tráfego de entrada quanto de saída.

## Como funciona o Network Layer Protection

Ao ativar o módulo **Network Layer Protection** nas **Main Settings** de um **Edge Firewall** Rule Set, as condições (*Criteria*) e os comandos (*Behavior*) serão ativados na aba **Rules Engine** do Edge Firewall.

> Saiba mais sobre [Edge Firewall](/pt-br/documentacao/produtos/edge-firewall/) e sobre [Edge Firewall Rule Sets](/pt-br/documentacao/produtos/edge-firewall/rules-engine/).

Os Criteria disponíveis com a ativação do módulo Network Layer Protection são:

- Hostname
- Network
- Request URI
- Scheme

E os Behaviors são:

- Deny (403 Forbidden)
- Drop (close without response)
- Set Rate Limit

A configuração de Network Layer Protection é feita na aba **Rules Engine** do Edge Firewall. Lá, você pode usar a criteria `Network` para criar regras com [Network Lists](/pt-br/documentacao/produtos/edge-firewall/network-layer-protection/network-lists/), com base na rede, localização do usuário ou ASN ou usar listas prontas para uso que são mantidas atualizadas pela própria Azion, como os endereços de saída da Rede Tor *Azion IP Tor Exit Nodes*. Você também pode monitorar e bloquear comportamentos suspeitos, ou até mesmo aplicar restrições de acordo com o behavior escolhido.

Ative todos os módulos de **Edge Firewall** para obter várias combinações de criteria e behavior no **Edge Firewall Rule Set**.

Exemplo de lógica de criteria e behavior utilizando **Network Layer Protection** e **Web Application Firewall**:

Criteria: `If Network matches My-Country-BlockList and Header User Agent does not match Googlebot`.

Behavior: `Then Deny (403 Forbidden)`.

Neste exemplo, as requisições originadas de países que estão na lista de bloqueio serão bloqueadas, a menos que o header `user-agent` contenha "Googlebot".

> As condições e comportamentos disponíveis na aba Rules Engine dependem dos módulos de Edge Firewall ativados.

## Network Lists

Por meio **de Network Lists**, você pode criar, pesquisar ou atualizar as Network Lists usadas no no **Rules Engine do Edge Firewall**.

Também é possível manter suas próprias **Network Lists** através do [Real-Time Manager (RTM)](https://manager.azion.com) ou pela [API](https://api.azion.com/). Uma única Network List pode ser associada a mais de um Rule Set de Edge Firewall.

Sempre que a Network List for atualizada, ela se propagará automaticamente para todas as regras associadas à ela.

Saiba mais sobre [Network Lists](/pt-br/documentacao/produtos/edge-firewall/network-layer-protection/network-lists/).

> Se você tiver um *Security Information and Event Management (SIEM)* ou outras ferramentas de segurança em sua infraestrutura, poderá utilizar as Network Lists para manter suas *blocklists* e *allowlists* sempre atualizadas.

## Origin Shield

**Origin Shield** é um complemento de **Network Layer Protection**. Com Origin Shield, você pode criar um perímetro de segurança para sua infraestrutura de origem, seja uma *nuvem*, seja de um provedor de *hospedagem* ou até mesmo seu próprio *data center*. Com este serviço, a sua origem poderá restringir o acesso apenas a endereços IP específicos da nossa rede e bloquear qualquer outro acesso à sua origem.

A lista de IPs da Azion pode mudar com frequência, mas depois de atualizada, os novos servidores só serão colocados em produção para aqueles usuários que utilizam Origin Shield há pelo menos 7 dias após a publicação da lista. Você também pode acompanhar e rastrear as alterações feitas na lista através do *histórico* do **RTM**. Lá, você pode encontrar quais IPs foram adicionados ou excluídos da lista.

O processo de *quarentena* permite que você adicione novos endereços com uma semana de antecedência, o que significa que sempre que um novo endereço for adicionado à lista, ele ficará em espera por 7 dias antes de ser ativado, dando a você tempo para atualizar suas regras no Edge Firewall.

> Todos os usuários Azion com o produto habilitado são notificados por email sempre que os endereços do Origin Shield mudam.
