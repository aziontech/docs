Você pode fazer o debug de requisições para as suas applications enviando o cabeçalho HTTP `Pragma: azion-debug-cache`. Este cabeçalho instrui os Edge Servers da Azion a retornarem indicadores detalhados sobre o estado do cache no cabeçalho da resposta.

---

## Verificando os Indicadores via cURL

A forma mais rápida e recomendada para verificar os indicadores é utilizar o `cURL` diretamente no terminal da sua máquina:

1. Abra o terminal.
2. Execute o comando enviando o cabeçalho `Pragma: azion-debug-cache` (substitua a URL de exemplo pela URL da sua aplicação):

```bash
curl -I -H "Pragma: azion-debug-cache" [https://www.exemplo.com](https://www.exemplo.com)

```

> **Dica:** O parâmetro `-I` solicita apenas os cabeçalhos da resposta HTTP, facilitando a leitura rápida dos dados de cache sem baixar o corpo da página.

3. Inspecione a resposta retornada. Os indicadores da Azion estarão listados sob os cabeçalhos que iniciam com `X-Cache`.

---

## Alternativa: Google Chrome DevTools (Nativo)

Se você preferir inspecionar os cabeçalhos através do navegador sem utilizar extensões de terceiros:

1. Abra o **Google Chrome** e acesse a página desejada.
2. Abra o **Developer Tools** pressionando `F12` (ou `Ctrl+Shift+I` / `Cmd+Option+I`).
3. Vá para a aba **Network**.
4. Recarregue a página (`F5`).
5. Selecione a requisição principal da lista e navegue até a aba **Headers**.
6. Procure pelos cabeçalhos de resposta (*Response Headers*) iniciados em `X-Cache`.

## Tabela de Indicadores de Cache

Ao enviar o cabeçalho de debug, a Azion retornará os seguintes campos na resposta HTTP:

| Campo | Descrição |
| --- | --- |
| X-Cache | Status da requisição, podendo ser HIT, MISS, EXPIRED, UPDATING ou STALE. |
| X-Cache-Key | Chave de cache do objeto. Normalmente utilizada a combinação de host + URI + argumentos. |
| X-Cache-File | Arquivo de cache do objeto, sendo ele composto do md5 da chave de cache. |
| X-Cache-Since | Unix Time Stamp* da entrada do objeto em cache. |
| X-Cache-Expire | Unix Time Stamp* da saída do objeto em cache. |
| X-Cache-Expire-In | Tempo restante do objeto em cache, em segundos. |
| X-Cache-Valid | Tempo configurado para o objeto no cache. Este tempo pode ser configurado incondicionalmente no servidor ou pode vir da origem via cabeçalhos Cache-Control. |
| X-Cache-Config | ID da configuração do virtual host no webserver. Este valor é gerado pelo Azion Console. |
| X-Cache-ID | Identificador único da requisição. |

** Você pode converter um Unix Time Stamp em formato humano utilizando [o site Unix Time Stamp](http://www.unixtimestamp.com/).*
