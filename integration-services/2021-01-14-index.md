# Servicios de **Integración**

[Edite no GitHub <svg width="14" height="14" xmlns="http://www.w3.org/2000/svg"><g fill="none" stroke="#F3652B"><path d="M4.81.71H.672v11.43H12.1V8.001" stroke-width=".8"/><path d="M6.87.786h5.155V5.94M6.31 6.5L12.026.786"/></g></svg>](https://github.com/aziontech/docs_en/edit/master/integration-services/2021-01-14-index.md)

Es la forma más rápida y segura de migrar tu contenido, aplicaciones web y API a Azion, sin importar qué tan complejo sea o la gran cantidad de recursos que poseas. Nuestro equipo de **Servicios de integración** habilitará los servicios que has contratado y te proporcionará las herramientas, la documentación y la capacitación necesarias para que uses con éxito la Plataforma de Azion.

#### El servicio de integración está disponible en las siguientes modalidades, diseñadas de acuerdo con las necesidades individuales de cada cliente:

> 1. [Integración estándar](#integracion-estandar)
> 2. [Integración gestionada](#integracion-gestionada)

---

## 1. Integración estándar{#integracion-estandar}

Recomendada para empresas con requisitos de implementación más simples y rápidos. Azion pondrá a disposición herramientas y documentación técnica para guiar el proceso de integración, así como un experto de Azion para dar soporte al cliente durante todo el proceso, que incluye:

1.  activación de los servicios Azion contratados según se describe en la Orden de servicio;
2.  validación de los requisitos del cliente;
3.  capacitación sobre el uso de [Real-Time Manager](https://manager.azion.com/);
4.  revisión y optimización de las configuraciones del servicio de Azion para el cliente.

#### El especialista de Azion seguirá un proceso de integración que consta de los siguientes pasos:

| Etapa                                                        | Responsabilidades del cliente                                | Responsabilidades de Azion                                   |
| ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 1. Asignación de recursos del proyecto, recopilación de requisitos, capacitación, revisión de las herramientas y la documentación | - Asignar los recursos necesarios para el proceso de integración y gestión de proyectos.<br/><br>- Revisión de documentación técnica y guías de implementación. | - Asignar un especialista en servicios de integración.<br/>- Programar una capacitación técnica por teléfono o videoconferencia sobre cómo utilizar la documentación de Real-Time Manager y de Azion. |
| 2.  Habilitación, pruebas e implementación de la configuración | - Preparar la infraestructura de origen para suministrar contenido a través de Azion.<br/><br>- Configurar Real-Time Manager.<br/><br>- Probar el entorno para asegurarse de que funciona y cumple con los requisitos.<br/>- Después de la aprobación, el DNS apunta a Azion, por lo que el contenido será suministrado por la Plataforma de Azion. | - Dar soporte al cliente acerca de preguntas sobre cómo configurar la Plataforma de Azion a través de Real-Time Manager.<br/><br> - Brindar las recomendaciones y las mejores prácticas para habilitar las configuraciones.<br/> - Revisar la configuración y los requisitos junto con el cliente.<br/><br> - Dar soporte al cliente de forma remota sobre cómo mover el contenido a la Plataforma de Azion, en una fecha mutuamente acordada, en horario comercial. |
| 3. Conclusión                                                | - Probar el contenido en producción.<br/>- Analizar registros y gráficos de analítica. | - Conducir junto con el cliente un proceso de revisión sobre los logros del proyecto.<br/><br>- Revisar procedimientos de soporte para incidentes y problemas de producción. |

> El servicio de  **Integración estándar** no incluye reuniones en las instalaciones del cliente.

---

## 2. Integración gestionada {#integracion-gestionada}

Azion ofrece el servicio de **Integración gestionada** para dar soporte al cliente durante todo el proceso de integración. Este servicio es idóneo para clientes con demandas más complejas, que necesitan soporte para atender todos los aspectos de sus requisitos técnicos y comerciales.

#### Un experto de Azion estará disponible para dar soporte al cliente durante todo el proceso a través de:

1.  la activación de los servicios de Azion contratados según lo descrito en la Orden de servicio;
2.  la recopilación de requisitos técnicos y comerciales junto con el cliente;
3.  la capacitación sobre el uso de [Real-Time Manager](https://manager.azion.com/);
4.  la configuración de los servicios de Azion para el cliente, y
5.  la revisión y optimización de configuraciones de servicio de acuerdo con las demandas del cliente.

#### El especialista de Azion seguirá un proceso de integración que consta de los siguientes pasos:

| Etapa                                                        | Responsabilidades del cliente                                | Responsabilidades de Azion                                   |
| ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 1. Asignación de recursos del proyecto, recopilación de requisitos, capacitación, revisión de herramientas y documentación | - Asignar los recursos necesarios para colaborar con el equipo de integración de Azion.<br/><br>- Respaldar internamente la recopilación de requisitos técnicos y comerciales conducidos por Azion, al responder preguntas y proporcionar información. | - Asignar un especialista en Integración.<br/><br>- Programar el inicio del proyecto con el cliente para la recopilación de requisitos.<br/><br>- Realizar un análisis detallado de las necesidades presentadas por el cliente, mediante la obtención de requerimientos técnicos y comerciales de las descripciones presentadas por el cliente o el análisis del contenido o aplicación. |
| 2. Planificación del proyecto                                | - Confirmar el cronograma del proyecto, el plan de implementación y el plan de pruebas.<br/> - Diseñar el plan de pruebas funcional y condiciones para las pruebas de aceptación. | - Identificar y diseñar las soluciones adecuadas para los requerimientos técnicos o comerciales de cada cliente.<br/><br>- Planificar la implementación y los cronogramas de pruebas.<br/><br>- Validar el plan de pruebas funcional y las condiciones para las pruebas de aceptación del cliente. |
| 3. Implementation                                            | - Preparar la infraestructura de origen para suministrar el contenido a través de la Plataforma de Azion. | - Realizar la configuración en Real-Time Manager para el cliente<br/><br>- Proporcionar la configuración necesaria para que el cliente haga pruebas.<br/><br> - Validar la configuración de origen ejecutada por el cliente, para la integración con Azion. |
| 4. Pruebas                                                   | -  Probar el contenido y la aplicación en el entorno proporcionado por Azion para asegurarse de que funciona.<br/><br>- Obtener la aceptación de las partes interesadas para el funcionamiento de la plataforma como se esperaba. | - Probar el contenido y la aplicación de acuerdo con los requisitos planteados durante la fase inicial del proyecto.<br/><br>- Probar el desempeño del suministro y el logro de los objetivos definidos. |
| 5. Operaciones                                               | - Ejecutar el contenido o la implementación de la aplicación para la Plataforma de Azion.<br/>- Probar el entorno de producción, realizar un seguimiento de los gráficos y los registros de acceso<br/><br>- Solicitar rollback en caso de problemas. | - Observar el comportamiento del entorno de producción para identificar cualquier anomalía<br/><br>- Dar soporte al cliente con lo que sea necesario durante la producción.<br/><br> - Seguimiento del proceso y de las pruebas del cliente. |
| 6. Conclusión                                                | - Validar los logros de los objetivos del proyecto.<br/><br>- Comunicar los procedimientos de Soporte al equipo responsable de administrar el contenido y la aplicación. | - Programar una reunión de revisión.<br/><br>- Conducir junto con el cliente la revisión y la validación del éxito del proyecto.<br/><br>- Examinar las necesidades no planteadas durante la planificación y realizar ajustes<br/> - Revisar los procedimientos de soporte sobre incidentes y problemas en producción.<br/><br>- Finalizar proyecto. |

> Todo el proceso de integración se realiza de forma remota, por *teléfono* o *videoconferencia*. 
>
> En caso de solicitarse, pueden llevarse a cabo reuniones presenciales en las instalaciones del cliente, pero los costos de viaje y alojamiento deberán ser cubiertos por el contratista.
